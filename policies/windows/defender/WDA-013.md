# WDA-013 — Disable Defender for Specific Server Roles (Performance Hardening)

**ID:** WDA-013  
**Category:** Windows Defender AV / Server Exclusions  
**Risk Level:** 🟠 Medium  
**OS:** Windows Server 2016+  
**Source:** Microsoft Server Role Documentation

---

## Description

On Windows Server, certain roles and processes should have targeted exclusions to prevent Defender from scanning files it cannot usefully inspect (like live database files or Hyper-V VHDs) or causing deadlocks with critical processes. These are **narrow, justified exclusions** — not broad folder exclusions.

> ⚠️ **CRITICAL:** These exclusions are for servers only. Never apply on workstations.

## Role-Based Exclusions (Justified)

### Active Directory Domain Controllers

```powershell
# AD database files
Add-MpPreference -ExclusionPath "C:\Windows\NTDS"
Add-MpPreference -ExclusionProcess "C:\Windows\System32\ntdsa.dll"
Add-MpPreference -ExclusionExtension ".dit", ".edb", ".sdb", ".log"
```

### SQL Server

```powershell
# SQL data and log files (live, locked by SQL process)
Add-MpPreference -ExclusionProcess "sqlservr.exe", "sqlagent.exe"
Add-MpPreference -ExclusionExtension ".mdf", ".ldf", ".ndf"
# DO NOT exclude entire SQL data directory
```

### Hyper-V

```powershell
# VHD/VHDX files — large binary, scanning causes I/O storm
Add-MpPreference -ExclusionExtension ".vhd", ".vhdx", ".avhd"
Add-MpPreference -ExclusionProcess "vmms.exe", "vmwp.exe"
```

### IIS Web Server

```powershell
# IIS process (NOT the web root — always scan web root)
Add-MpPreference -ExclusionProcess "w3wp.exe"
# NEVER exclude wwwroot or web content directories
```

## Exclusion Hygiene Rules for Servers

```
✅ Exclude specific process binaries, not entire service folders
✅ Exclude specific file extensions for database roles only
✅ Document every exclusion with business justification
❌ Never exclude C:\Windows\Temp on servers
❌ Never exclude C:\Windows\System32
❌ Never exclude wwwroot or web content paths
❌ Never exclude script interpreter paths (cscript.exe, powershell.exe)
```
