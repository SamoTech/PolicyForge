# WDA-011 — Configure Signature Update Frequency

**ID:** WDA-011  
**Category:** Windows Defender AV / Signature Management  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** Microsoft Security Baseline

---

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Signature Updates
  SignatureUpdateInterval         = 4    (hours between update checks, 0=disable)
  SignatureUpdateCatchupInterval  = 1    (days before forcing catchup update)
  ForceUpdateFromMU               = 1    (use Microsoft Update as fallback)
  FallbackOrder                   = "MicrosoftUpdateServer|MMPC"  (update sources)
  DisableScheduledSignatureUpdateOnBattery = 0  (update even on battery)
  RealtimeSignatureDelivery       = 1    (deliver critical signatures immediately)
```

## PowerShell

```powershell
# Check signature age
Get-MpComputerStatus | Select `
    AntivirusSignatureLastUpdated,
    AntivirusSignatureAge,
    AntivirusSignatureVersion,
    AntispywareSignatureAge

# Force immediate signature update
Update-MpSignature -UpdateSource MicrosoftUpdateServer

# Alert if signatures older than 2 days (monitoring script)
$sigAge = (Get-MpComputerStatus).AntivirusSignatureAge
if ($sigAge -gt 2) {
    Write-Warning "ALERT: Defender signatures are $sigAge days old!"
    # Send to SIEM / monitoring platform
}
```

## Signature Update Sources (Priority Order)

```
1. WSUS / SCCM (internal — fastest, controlled)
2. Microsoft Update (fallback for internet-connected)
3. Microsoft Malware Protection Center (MMPC)
4. UNC share (air-gapped environments)
5. Internal UNC (\\server\DefenderUpdates)
```
