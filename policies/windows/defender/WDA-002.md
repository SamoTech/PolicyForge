# WDA-002 — Enable Cloud-Delivered Protection (MAPS)

**ID:** WDA-002  
**Category:** Windows Defender AV / Cloud Protection / MAPS  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Microsoft Defender Antivirus
                 └─ MAPS
                      ├─ Join Microsoft MAPS: Enabled → Advanced MAPS
                      ├─ Send file samples when further analysis is required: Enabled → Send safe samples
                      ├─ Configure the 'Block at First Sight' feature: Enabled
                      └─ Configure extended cloud check: Enabled → 50 seconds
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Spynet
  SpynetReporting              = 2   (0=off, 1=basic, 2=advanced)
  SubmitSamplesConsent         = 1   (0=always prompt, 1=safe samples, 2=never, 3=all)

HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\MpEngine
  MpCloudBlockLevel            = 2   (0=default, 1=moderate, 2=high, 4=high+, 6=zero tolerance)
  MpBafsExtendedTimeout        = 50  (cloud check timeout in seconds, max 60)

HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection
  DisableBlockAtFirstSeen      = 0   (0=enabled, 1=disabled)
```

## PowerShell

```powershell
# Verify cloud protection settings
Get-MpPreference | Select `
    MAPSReporting,
    SubmitSamplesConsent,
    CloudBlockLevel,
    CloudExtendedTimeout,
    DisableBlockAtFirstSeen

# Check cloud protection status
Get-MpComputerStatus | Select `
    AntispywareEnabled,
    CloudProtectionEnabled
```

## Block at First Sight Explained

```
Traditional AV flow:
  File arrives → Check local signatures → Unknown → Allow (RISK)

Block at First Sight flow:
  File arrives → Check local signatures → Unknown
    → Send metadata hash to cloud → Cloud checks global reputation
    → Unknown in cloud → Block file → Submit sample → Analyze (50s)
    → If malicious: block permanently
    → If clean: release

Result: Zero-day malware blocked even before signature exists
```

## MpCloudBlockLevel Values

| Value | Level | Behavior |
|---|---|---|
| 0 | Default | Microsoft default blocking |
| 1 | Moderate | Block high-confidence malware |
| **2** | **High** | **Block suspicious + high-confidence** |
| 4 | High+ | Aggressive — more false positives |
| 6 | Zero Tolerance | Block ALL unknowns — kiosks/air-gapped only |

## Impact

- ✅ Zero-day threats blocked before signatures exist
- ✅ File hashes sent to Microsoft for global threat intelligence
- ⚠️ Requires internet connectivity to Microsoft cloud endpoints
- ⚠️ Air-gapped systems: use MpCloudBlockLevel = 6 + offline signature updates
- ℹ️ SubmitSamplesConsent = 1 (safe samples only) — no sensitive documents sent
