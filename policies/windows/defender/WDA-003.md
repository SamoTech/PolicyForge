# WDA-003 — Enable Tamper Protection

**ID:** WDA-003  
**Category:** Windows Defender AV / Tamper Protection  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1903+, Windows 11  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Microsoft Defender Antivirus
                 └─ Features
                      └─ Configure Microsoft Defender Tamper Protection: Enabled
```

## Registry

```
; Tamper Protection is primarily enforced via Windows Security Center
; and optionally via Intune/MEM — not via traditional GPO registry write
; The registry key below is READ-ONLY when Tamper Protection is active:
HKLM\SOFTWARE\Microsoft\Windows Defender\Features
  TamperProtection = 5   (5=on, 4=off)
; Attempting to write this key while TP is enabled will be blocked
```

## Intune / MDE Deployment (Recommended)

```
Microsoft Endpoint Manager > Endpoint Security > Antivirus
  └─ Windows Security experience
       └─ Enable tamper protection to prevent Microsoft Defender
            Antivirus from being disabled: Enabled
```

## PowerShell

```powershell
# Check Tamper Protection status
Get-MpComputerStatus | Select IsTamperProtected, TamperProtectionSource

# Windows Security Center check
$tp = (Get-MpComputerStatus).IsTamperProtected
if ($tp) {
    Write-Host "[OK] Tamper Protection: ENABLED" -ForegroundColor Green
} else {
    Write-Host "[FAIL] Tamper Protection: DISABLED" -ForegroundColor Red
    Write-Host "Action: Enable via Intune/MDE or Windows Security Center"
}
```

## What Tamper Protection Blocks

```
Without Tamper Protection (attacker can):
  reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware /t REG_DWORD /d 1
  Set-MpPreference -DisableRealtimeMonitoring $true
  net stop WinDefend
  sc config WinDefend start=disabled
  → Defender silently disabled. Malware runs undetected.

With Tamper Protection enabled:
  All above commands → ACCESS DENIED
  Even SYSTEM and local administrator cannot disable Defender
  Only Intune/MDE cloud console can modify settings
  Event 5013 logged: Tamper Protection blocked a change
```

## Tamper Protection Event IDs

| Event ID | Meaning | Action |
|---|---|---|
| 5013 | Tamper Protection blocked a change | Investigate who tried |
| 5001 | Real-time protection disabled | P1 — Tamper Protection may be off |
| 5004 | Real-time protection config changed | Review change |

## Impact

- ✅ Defender cannot be disabled by malware, scripts, or local admins
- ✅ Registry-based AV disable attacks blocked
- ✅ Service-stop attacks blocked
- ⚠️ GPO cannot override Tamper Protection when enabled via Intune
- ℹ️ Manage via MDE/Intune for enterprise-wide enforcement
