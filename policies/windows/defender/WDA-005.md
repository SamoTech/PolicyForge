# WDA-005 — Enable PUA (Potentially Unwanted Application) Blocking

**ID:** WDA-005  
**Category:** Windows Defender AV / PUA Protection  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10 2004+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Microsoft Defender Antivirus
                 └─ Configure detection for potentially unwanted applications:
                      Enabled → Block
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender
  PUAProtection = 1   (0=off, 1=block, 2=audit only)
```

## PowerShell

```powershell
# Enable PUA protection
Set-MpPreference -PUAProtection Enabled

# Verify
Get-MpPreference | Select PUAProtection
# 1 = Enabled, 2 = AuditMode, 0 = Disabled

# Check PUA detections in last 7 days
Get-MpThreatDetection | Where-Object { $_.Resources -match "PUA" } |
    Select ThreatName, ActionSuccess, InitialDetectionTime
```

## What PUA Protection Blocks

| Category | Examples | Risk |
|---|---|---|
| **Cryptominers** | XMRig, NiceHash miners | CPU theft, persistence |
| **Adware** | OpenCandy, Conduit | User tracking, browser hijack |
| **Bundled software** | Toolbars, search hijackers | Unwanted system changes |
| **Hack tools** | Password crackers, keygens | Policy violation |
| **Evasion tools** | Process hollowers | Pre-malware staging |
| **Credential harvesters** | Some "password recovery" tools | Data theft |

## Impact

- ✅ Blocks cryptominer downloads and installation
- ✅ Blocks common enterprise policy violations (cracks, keygens)
- ⚠️ Some legitimate dual-use tools may be flagged (IT tools like nirsoft)
- ℹ️ Start with PUAProtection = 2 (audit) to identify false positives before enforcing
