# WDA-004 — Harden Defender Exclusions (Anti-Exclusion Policy)

**ID:** WDA-004  
**Category:** Windows Defender AV / Exclusion Hardening  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** Microsoft Security Research · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Microsoft Defender Antivirus
                 └─ Exclusions
                      ├─ Control Configured Local Administrator overrides
                      │    for settings listed in the Exclusions subsetting: Disabled
                      └─ Turn off Auto Exclusions: Disabled  (keep auto exclusions for server roles)
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Exclusions
  DisableLocalAdminMerge = 1   (1=GPO only, ignore local exclusions)

; Document all approved exclusions here (GPO-managed):
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Exclusions\Paths
  ; Only add after documented business justification + security review
  ; Example: C:\Program Files\SpecificApp\scanner.exe (NOT entire folder)

HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Exclusions\Extensions
  ; Almost never justified — extension exclusions are high-risk
  ; Attackers rename malware to excluded extensions (.log, .dat)
```

## PowerShell — Audit Current Exclusions

```powershell
# List all current Defender exclusions
$prefs = Get-MpPreference
Write-Host "=== PATH EXCLUSIONS ==="
$prefs.ExclusionPath | ForEach-Object { Write-Host "  $_" }

Write-Host "`n=== EXTENSION EXCLUSIONS ==="
$prefs.ExclusionExtension | ForEach-Object { Write-Host "  $_" }

Write-Host "`n=== PROCESS EXCLUSIONS ==="
$prefs.ExclusionProcess | ForEach-Object { Write-Host "  $_" }

Write-Host "`n=== LOCAL ADMIN MERGE ==="
Write-Host "  DisableLocalAdminMerge: $(Get-ItemProperty 'HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\Exclusions' -Name DisableLocalAdminMerge -ErrorAction SilentlyContinue)"
```

## The Exclusion Attack Vector

Exclusions are the #1 way attackers and malware disable Defender without triggering tamper protection alerts:

```
Attack Scenario 1: Attacker adds path exclusion
  reg add "HKLM\SOFTWARE\Microsoft\Windows Defender\Exclusions\Paths"
    /v "C:\ProgramData\malware" /t REG_DWORD /d 0
  → Drops malware in excluded path → Defender ignores it

Attack Scenario 2: Ransomware adds extension exclusion
  Set-MpPreference -ExclusionExtension ".encrypted"
  → Encrypts files to .encrypted extension → No detection

Attack Scenario 3: LOLBin-based exclusion
  wmic /namespace:\\root\Microsoft\Windows\Defender
    path MSFT_MpPreference call Add PathExclusions=@("C:\Windows\Temp")
  → Executes payload from Temp without detection

Defense:
  DisableLocalAdminMerge = 1 → Only GPO-managed exclusions apply
  Tamper Protection → Blocks exclusion modification via registry
  GPO audit: review exclusions quarterly
```

## Exclusion Hygiene Rules

```
✅ Always: Exclude specific .exe files, not entire folders
✅ Always: Document business justification for each exclusion
✅ Always: Review exclusions quarterly
❌ Never: Exclude entire drives (C:\, D:\)
❌ Never: Exclude user-writable folders (C:\Temp, C:\Users\*\AppData)
❌ Never: Exclude file extensions (.exe, .dll, .ps1, .js)
❌ Never: Exclude without security team approval
```
