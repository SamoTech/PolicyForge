# WDA-006 — Enable Network Protection

**ID:** WDA-006  
**Category:** Windows Defender AV / Network Protection  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1709+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Microsoft Defender Exploit Guard
                 └─ Network Protection
                      └─ Prevent users and apps from accessing dangerous websites:
                           Enabled → Block
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Windows Defender Exploit Guard\Network Protection
  EnableNetworkProtection = 1   (0=off, 1=block, 2=audit)
```

## PowerShell

```powershell
# Enable Network Protection
Set-MpPreference -EnableNetworkProtection Enabled

# Verify
Get-MpPreference | Select EnableNetworkProtection
# 1 = Enabled (Block mode)

# Check network protection events
Get-WinEvent -LogName "Microsoft-Windows-Windows Defender/Operational" |
    Where-Object { $_.Id -in @(1125, 1126, 1127, 1128) } |
    Select TimeCreated, Id, Message | Select-Object -First 20
```

## What Network Protection Blocks

```
Network Protection extends SmartScreen to the OS level (not just Edge):

✔ Blocks outbound connections to:
   • Known phishing domains
   • Malware C2 servers
   • Exploit kit hosting sites
   • Low-reputation IP addresses

✔ Works for ALL applications:
   • Chrome, Firefox (not just Edge)
   • PowerShell Invoke-WebRequest
   • Malware calling home
   • cmd.exe curl/wget equivalents

✔ Event IDs:
   1125 — Connection blocked (Block mode)
   1126 — Connection would be blocked (Audit mode)
```

## Impact

- ✅ C2 callback blocking for all processes (not just browser)
- ✅ Complements Edge SmartScreen for non-Edge traffic
- ⚠️ Start in Audit mode (= 2) to identify legitimate traffic
- ⚠️ Some corporate proxy/VPN setups may need allow-list configuration
