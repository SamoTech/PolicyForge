# WDA-007 — Enable Controlled Folder Access (Anti-Ransomware)

**ID:** WDA-007  
**Category:** Windows Defender AV / Ransomware Protection  
**Risk Level:** 🟠 High  
**OS:** Windows 10 1709+, Windows 11  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Microsoft Defender Exploit Guard
                 └─ Controlled Folder Access
                      ├─ Configure Controlled folder access: Enabled → Enabled
                      ├─ Configure protected folders: Enabled → [list]
                      └─ Configure allowed applications: Enabled → [allowlist]
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Windows Defender Exploit Guard\Controlled Folder Access
  EnableControlledFolderAccess = 1   (0=off, 1=block, 2=audit, 3=block+audit disk)

; Protected folders (in addition to defaults: Desktop, Documents, Pictures, Music, Videos)
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Windows Defender Exploit Guard\Controlled Folder Access\ProtectedFolders
  1 = "C:\Finance\"
  2 = "C:\HR\"
  3 = "C:\Confidential\"

; Allowed applications (must explicitly whitelist if blocked)
HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Windows Defender Exploit Guard\Controlled Folder Access\AllowedApplications
  1 = "C:\Program Files\Notepad++\notepad++.exe"
```

## PowerShell

```powershell
# Enable Controlled Folder Access
Set-MpPreference -EnableControlledFolderAccess Enabled

# Add protected folders
Add-MpPreference -ControlledFolderAccessProtectedFolders "C:\Finance"
Add-MpPreference -ControlledFolderAccessProtectedFolders "C:\Confidential"

# Allow specific applications
Add-MpPreference -ControlledFolderAccessAllowedApplications \
    "C:\Program Files\Notepad++\notepad++.exe"

# Verify
Get-MpPreference | Select EnableControlledFolderAccess,
    ControlledFolderAccessProtectedFolders, ControlledFolderAccessAllowedApplications

# Check CFA events (blocked access attempts)
Get-WinEvent -LogName "Microsoft-Windows-Windows Defender/Operational" |
    Where-Object { $_.Id -eq 1123 } |
    Select TimeCreated, Message | Select-Object -First 20
```

## Ransomware Protection Flow

```
Ransomware execution attempt:
  1. Ransomware (evil.exe) tries to write encrypted files to C:\Documents
  2. CFA detects: untrusted process writing to protected folder
  3. CFA blocks the write operation → ACCESS DENIED
  4. Event 1123 logged: "Blocked access to protected folder"
  5. Ransomware cannot encrypt protected files

Allowed process flow:
  1. Word.exe tries to save to C:\Documents
  2. CFA: Word.exe is signed Microsoft app → ALLOWED
  3. File saved normally
```

## CFA Deployment Strategy

```
Phase 1: Audit mode (= 2)
  → Run for 1-2 weeks
  → Review Event 1124 (would have been blocked)
  → Add legitimate apps to allowlist

Phase 2: Block mode (= 1)
  → Enable after allowlist is built
  → Monitor Event 1123 (actual blocks)
  → Handle helpdesk tickets for legitimate app issues
```

## Default Protected Folders

Desktop, Documents, Downloads, Pictures, Music, Videos, Favorites, and OneDrive sync folders are protected by default.

## Impact

- ✅ Ransomware cannot encrypt protected folders
- ✅ Unauthorized processes cannot write to sensitive paths
- ⚠️ High false positives on first deployment — must build allowlist first
- ⚠️ Custom/legacy applications may need explicit allowlisting
