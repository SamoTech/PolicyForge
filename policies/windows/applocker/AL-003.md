# AL-003 — AppLocker Script Rules (Block PowerShell/CMD from User Paths)

**ID:** AL-003  
**Category:** AppLocker / Script Rules  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 Enterprise/Education, Windows 11  
**Source:** Microsoft AppLocker Documentation

---

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Application Control Policies
                 └─ AppLocker
```

## Covered Script Types

```
.ps1   PowerShell scripts
.cmd   Command scripts
.bat   Batch files
.vbs   VBScript
.js    JScript
.wsf   Windows Script Files
```

## PowerShell

```powershell
# Generate default Script rules baseline
Get-AppLockerPolicy -Local | Format-List

# Test a specific script against the policy
Test-AppLockerPolicy -Path "C:\Users\user\Downloads\payload.ps1" -User "Everyone"

# Get all Script rule violations in last 24 hours
Get-WinEvent -LogName "Microsoft-Windows-AppLocker/Script and Interpreted" -MaxEvents 100 |
    Where-Object { $_.Id -in @(8005, 8006, 8003, 8004) } |
    Select TimeCreated, Id, Message
```

## Description

Script rules block execution of script files (.ps1, .vbs, .js, .bat, .cmd, .wsf) from user-writable paths. Script-based attacks — especially PowerShell and JavaScript — are among the most common malware delivery methods. Blocking scripts from %TEMP%, %APPDATA%, Desktop, and Downloads removes the most dangerous execution paths while preserving legitimate scripts deployed to system directories.

## Impact

- ✅ Blocks .ps1/.vbs/.js dropped in user temp/download paths (major malware vector)
- ✅ Works alongside ASR-006 (block JS/VBS launcher) for layered defense
- ⚠️ Blocks user-deployed PowerShell scripts not in system paths
- ⚠️ Some admin tools using portable scripts from user paths will be affected
- ℹ️ Combine with PowerShell Constrained Language Mode for deeper PowerShell control
