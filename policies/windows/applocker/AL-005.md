# AL-005 — AppLocker Windows Installer Rules

**ID:** AL-005  
**Category:** AppLocker / MSI Rules  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 Enterprise/Education, Windows 11  
**Source:** Microsoft AppLocker Documentation

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Application Control Policies
                 └─ AppLocker
                      └─ Windows Installer Rules
                           └─ Create Default Rules + enforce
```

## PowerShell

```powershell
# Test whether an MSI file would be allowed
Test-AppLockerPolicy -Path "C:\Downloads\suspicious-installer.msi" -User "domain\standarduser"

# Get all MSI-related AppLocker events
Get-WinEvent -LogName "Microsoft-Windows-AppLocker/MSI and Script" |
    Where-Object { $_.Id -in @(8005, 8006, 8007, 8008) } |
    Select TimeCreated, Id, Message | Format-Table -AutoSize

# Generate rules from a reference machine's installed MSIs
Get-AppLockerFileInformation -Packages | New-AppLockerPolicy -RuleType Publisher -User "Everyone"
```

## Default Rules

```
Rule 1: Allow Everyone — Digitally signed Windows Installer files
Rule 2: Allow Everyone — %WINDIR%\Installer\*
Rule 3: Allow Administrators — All Windows Installer files
```

## Description

Windows Installer (MSI/MSP/MST) rules control which installer packages can run. Without MSI rules enforced, a standard user can run any .msi installer — including malware delivered as a signed or unsigned MSI package. MSI files are increasingly used in phishing campaigns because many security tools skip scanning them. Restricting MSI execution to signed files or specific publishers significantly reduces the attack surface.

## Impact

- ✅ Blocks unsigned/untrusted MSI packages from running
- ✅ Stops MSI-based malware delivery (increasingly common vector)
- ⚠️ Blocks unsigned third-party MSI installers
- ℹ️ Combine with Software Restriction Policy for older Windows versions
