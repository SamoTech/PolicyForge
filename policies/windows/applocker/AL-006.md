# AL-006 — AppLocker Packaged App (MSIX / Store App) Rules

**ID:** AL-006  
**Category:** AppLocker / Packaged Apps / MSIX  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10 1803+, Windows 11  
**Source:** Microsoft AppLocker Documentation

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Application Control Policies
                 └─ AppLocker
                      └─ Packaged App Rules
                           └─ Create Default Rules
```

## PowerShell

```powershell
# List all installed packaged apps on the reference machine
Get-AppxPackage -AllUsers | Select Name, Publisher, PackageFullName | Format-Table -AutoSize

# Generate AppLocker rules from installed packages
Get-AppLockerFileInformation -Packages |
    New-AppLockerPolicy -RuleType Publisher -User "Everyone" -RuleNamePrefix "Store" |
    Out-File C:\PackagedAppsPolicy.xml

# Block a specific Store app
# (Use deny rule targeting specific package publisher + name)

# Audit mode for packaged apps
Get-WinEvent -LogName "Microsoft-Windows-AppLocker/Packaged app-Execution" |
    Where-Object { $_.Id -in @(8020, 8021, 8022, 8023) } |
    Select TimeCreated, Id, Message
```

## Description

Packaged App rules control execution of MSIX and Store (UWP) apps. If you enforce Executable rules without configuring Packaged App rules, all Store/MSIX apps may be blocked by default. This policy ensures Store apps used in the enterprise (Teams, Calculator, Photos) are explicitly allowed, while providing a mechanism to block unwanted Store apps (gaming apps, social media) that arrive via Windows Update or user installation.

## Impact

- ✅ Prevents unauthorized Store app installation and execution
- ✅ Allows approved enterprise apps (Teams, Office MSIX, etc.)
- ⚠️ If Exe rules are enforced and Packaged App rules unconfigured — Store apps break
- ℹ️ Default rules allow all signed Microsoft apps — start there, then restrict

## Event IDs

| Event ID | Meaning |
|---|---|
| 8020 | Packaged app allowed (Audit) |
| 8021 | Packaged app blocked (Audit) |
| 8022 | Packaged app allowed (Enforce) |
| 8023 | Packaged app blocked (Enforce) |
