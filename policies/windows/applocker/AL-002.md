# AL-002 — Configure AppLocker Default Executable Allow Rules

**ID:** AL-002  
**Category:** AppLocker / Executable Rules  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 Enterprise/Education, Windows 11  
**Source:** Microsoft AppLocker Documentation

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Application Control Policies
                 └─ AppLocker
                      └─ Executable Rules
                           └─ Create Default Rules
```

## Default Rules Created

```
Rule 1: Allow Everyone — %PROGRAMFILES%\*     (Publisher or Path)
Rule 2: Allow Everyone — %WINDIR%\*           (Publisher or Path)
Rule 3: Allow Administrators — *              (All files)
```

## PowerShell

```powershell
# Generate default AppLocker executable rules
$policy = Get-AppLockerPolicy -Local

# Create default rules for Exe collection
$defaultRules = New-Object -TypeName Microsoft.Security.ApplicationId.PolicyManagement.PolicyModel.AppLockerPolicy
[Microsoft.Security.ApplicationId.PolicyManagement.PolicyHelper]::getDefaultRuleCollections()

# Export current effective policy
Get-AppLockerPolicy -Effective -Xml | Out-File C:\AppLockerPolicy.xml

# Import/merge a policy
Set-AppLockerPolicy -XmlPolicy C:\AppLockerPolicy.xml -Merge

# Set Exe collection to Enforce
# (Must be done in GPO: Action = Enforce Rules)

# Test which apps would be blocked before enforcing
Get-AppLockerPolicy -Effective | Test-AppLockerPolicy -Path "C:\Tools\custom.exe" -User "domain\user"
```

## XML Policy Template

```xml
<AppLockerPolicy Version="1">
  <RuleCollection Type="Exe" EnforcementMode="AuditOnly">
    <!-- Allow Windows directory -->
    <FilePathRule Id="921cc481-6e17-4653-8f75-050b80acca20"
                  Name="Allow Windows" Description="All files in Windows folder"
                  UserOrGroupSid="S-1-1-0" Action="Allow">
      <Conditions>
        <FilePathCondition Path="%WINDIR%\*"/>
      </Conditions>
    </FilePathRule>
    <!-- Allow Program Files -->
    <FilePathRule Id="a61c8b2c-a23a-44de-b2d7-b27a41b4e8e8"
                  Name="Allow Program Files" Description="All files in Program Files"
                  UserOrGroupSid="S-1-1-0" Action="Allow">
      <Conditions>
        <FilePathCondition Path="%PROGRAMFILES%\*"/>
      </Conditions>
    </FilePathRule>
    <!-- Allow Admins everything -->
    <FilePathRule Id="fd686d83-a829-4351-8ff4-27c7de5755d2"
                  Name="Allow Admins" Description="Administrators can run everything"
                  UserOrGroupSid="S-1-5-32-544" Action="Allow">
      <Conditions>
        <FilePathCondition Path="*"/>
      </Conditions>
    </FilePathRule>
  </RuleCollection>
</AppLockerPolicy>
```

## Description

The AppLocker default executable rules form the baseline allowlist for EXE and COM files. They permit execution from Windows and Program Files directories (where only admins can write) while implicitly denying execution from user-writable locations like %TEMP%, %APPDATA%, and Desktop — the most common malware staging paths. Start with default rules before adding publisher-based rules for specific applications.

## Impact

- ✅ Blocks execution from %TEMP%, %APPDATA%, Desktop, Downloads (malware staging paths)
- ✅ Allows all signed Windows and installed applications
- ⚠️ Blocks portable/unsigned apps not in Program Files
- ⚠️ Some legitimate tools run from user-writable paths will be blocked
- ℹ️ Always start in AuditOnly mode — review Event 8003/8004 before enforcing

## Event IDs to Monitor

| Event ID | Log | Meaning |
|---|---|---|
| 8003 | AppLocker/EXE and DLL | Would have been blocked (Audit mode) |
| 8004 | AppLocker/EXE and DLL | Was blocked (Enforce mode) |
| 8005 | AppLocker/EXE and DLL | Would have been allowed |
| 8006 | AppLocker/EXE and DLL | Was allowed |
