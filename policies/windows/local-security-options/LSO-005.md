# LSO-005 — Limit Cached Logon Credentials

**ID:** LSO-005  
**Category:** Local Security Options / Credential Caching  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      └─ Interactive logon: Number of previous logons to cache
                           (in case domain controller is not available): 4
```

## Registry

```
HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
  CachedLogonsCount = "4"   ; String value (not DWORD)
  ; 0 = no caching (only if always online)
  ; 4 = CIS recommendation
  ; Default = 10
```

## How Cached Credentials Are Stored & Attacked

```
Windows cached credential storage:
  Location: HKLM\SECURITY\Cache (requires SYSTEM access)
  Format: DCC2 (Domain Cached Credentials v2)
  Hash: PBKDF2-SHA1 — mscash2 format

Attack (requires SYSTEM/SeDebugPrivilege):
  secretsdump.py -hashes ... target/Administrator@192.168.1.1
  cachedump tool
  Mimikatz: lsadump::cache

Cracking DCC2:
  hashcat -m 2100 dcc2_hash.txt wordlist.txt
  DCC2 is SLOW to crack (PBKDF2 with 10,240 iterations)
  Weak passwords still vulnerable
  Strong 14+ char passwords: effectively uncrackable

Why limit to 4 (not 0):
  0 = users cannot log in if DC unreachable (laptops, travel)
  4 = allows offline logon while minimizing attack surface
  10 (default) = too many cached hashes exposed if disk stolen
```

## Disk Encryption Context

```
With BitLocker (see BL-001) + CachedLogonsCount = 4:
  Stolen laptop scenario:
  → BitLocker: cannot read disk without TPM/PIN
  → Even if BitLocker bypassed: only 4 DCC2 hashes
  → DCC2 slow to crack
  → Strong passwords: secure even if hashes extracted

Without BitLocker + CachedLogonsCount = 10:
  → Remove drive, mount on attacker machine
  → Extract SECURITY hive
  → Dump 10 domain account DCC2 hashes
  → Crack weak passwords offline
```
