# LSO-008 — Secure Channel & Domain Member Security

**ID:** LSO-008  
**Category:** Local Security Options / Domain Security  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Domain member: Digitally encrypt or sign secure channel data (always): Enabled
                      ├─ Domain member: Digitally encrypt secure channel data (when possible): Enabled
                      ├─ Domain member: Digitally sign secure channel data (when possible): Enabled
                      ├─ Domain member: Disable machine account password changes: Disabled
                           (i.e. password changes ARE enabled)
                      ├─ Domain member: Maximum machine account password age: 30 days
                      └─ Domain member: Require strong (Windows 2000 or later) session key: Enabled
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Services\Netlogon\Parameters
  RequireSignOrSeal        = 1   ; Encrypt or sign always
  SealSecureChannel        = 1   ; Encrypt when possible
  SignSecureChannel        = 1   ; Sign when possible
  DisablePasswordChange    = 0   ; Allow machine account password changes
  MaximumPasswordAge       = 30  ; Machine account password age (days)
  RequireStrongKey         = 1   ; Require 128-bit session key
```

## Description

Secure channel communications are the Netlogon RPC channel between domain members and domain controllers — used for authentication, policy retrieval, and machine account operations. Signing and encrypting this channel prevents:

- **Man-in-the-middle attacks** on domain authentication
- **Netlogon spoofing** (CVE-2020-1472 Zerologon class attacks)
- **Credential interception** during domain logon

## Machine Account Password Rotation

```
DisablePasswordChange = 0 (change enabled):
  Machine account password rotates every 30 days automatically
  → Even if machine account hash extracted, expires in ≤30 days
  → Limits Silver Ticket attack window

DisablePasswordChange = 1 (change disabled):
  → Never rotate → machine account hash valid indefinitely
  → Silver Ticket attacks persistent
  → Never disable unless required for specific service
```
