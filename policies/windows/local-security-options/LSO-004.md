# LSO-004 — SMB Signing (Client & Server)

**ID:** LSO-004  
**Category:** Local Security Options / SMB Security  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Microsoft network client: Digitally sign communications (always): Enabled
                      ├─ Microsoft network client: Digitally sign communications (if server agrees): Enabled
                      ├─ Microsoft network server: Digitally sign communications (always): Enabled
                      └─ Microsoft network server: Digitally sign communications (if client agrees): Enabled
```

## Registry

```
; SMB Client signing
HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters
  RequireSecuritySignature = 1   ; Always require signing
  EnableSecuritySignature  = 1   ; Sign if server agrees

; SMB Server signing
HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters
  RequireSecuritySignature = 1   ; Always require signing
  EnableSecuritySignature  = 1   ; Sign if client agrees
```

## PowerShell

```powershell
# Check SMB signing
Get-SmbServerConfiguration | Select RequireSecuritySignature, EnableSecuritySignature
Get-SmbClientConfiguration | Select RequireSecuritySignature, EnableSecuritySignature

# Enable server signing
Set-SmbServerConfiguration -RequireSecuritySignature $true -EnableSecuritySignature $true -Force

# Enable client signing
Set-SmbClientConfiguration -RequireSecuritySignature $true -EnableSecuritySignature $true -Force

# Check active SMB sessions and signing status
Get-SmbSession | Select ClientComputerName, ClientUserName, NumOpens, Dialect, SigningEnabled
```

## NTLM Relay Attack (Blocked by SMB Signing)

```
NTLM Relay without SMB signing:
  1. Attacker poisons LLMNR/NBT-NS (Responder)
  2. Victim authenticates to attacker (NTLM challenge)
  3. Attacker relays authentication to file server
  4. Attacker gains authenticated SMB session
  5. Can read files, execute commands (ntlmrelayx.py)

With SMB signing required:
  Step 3 → File server requires signed SMB
  Attacker cannot forge valid signature without session key
  → NTLM relay fails completely

Real tools blocked: ntlmrelayx.py, Responder relay, PetitPotam
```

## Windows 11 24H2 Default Change

Microsoft **enabled SMB signing by default** on Windows 11 24H2 for all connections. However, explicit GPO enforcement ensures this cannot be disabled by local configuration and applies to older OS versions in the fleet.

## Impact

- ✅ NTLM relay attacks completely blocked
- ✅ SMB man-in-the-middle attacks blocked
- ⚠️ ~10-15% performance overhead on very high SMB I/O workloads
- ⚠️ Older NAS/storage appliances may not support signing — audit before enforcing
