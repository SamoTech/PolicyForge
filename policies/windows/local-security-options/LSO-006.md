# LSO-006 — Interactive Logon: Warnings, Last Username & CTRL+ALT+DEL

**ID:** LSO-006  
**Category:** Local Security Options / Interactive Logon  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Interactive logon: Do not display last user name: Enabled
                      ├─ Interactive logon: Do not require CTRL+ALT+DEL: Disabled
                           (i.e. CTRL+ALT+DEL IS required)
                      ├─ Interactive logon: Message title for users attempting to log on:
                      │    "Authorized Use Only"
                      └─ Interactive logon: Message text for users attempting to log on:
                           [Legal banner text]
```

## Registry

```
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System
  DontDisplayLastUserName   = 1   ; Don't show last logged-on user
  DisableCAD                = 0   ; 0 = CTRL+ALT+DEL required (1 = not required)
  LegalNoticeCaption        = "Authorized Use Only"
  LegalNoticeText           = "This system is for authorized use only. Unauthorized access is prohibited and may be subject to prosecution."

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
  DontDisplayLastUserName   = 1
  DontDisplayLockedUserId   = 3   ; 3=don't display user info on locked screen
```

## Why Each Setting Matters

| Setting | Without Policy | With Policy | Security Gain |
|---|---|---|---|
| Last username shown | Attacker knows valid username | Hidden | Removes recon target |
| No CTRL+ALT+DEL | Any process can fake logon screen | Trusted path required | Stops password harvesting trojans |
| No legal banner | No legal standing for prosecution | Explicit warning | Legal protection |

## CTRL+ALT+DEL (Trusted Path) Explained

```
Without CTRL+ALT+DEL requirement:
  Malware can display a fake Windows logon screen
  → User types password into fake screen
  → Credentials harvested

With CTRL+ALT+DEL required:
  CTRL+ALT+DEL is a "secure attention sequence" intercepted by
  the Windows kernel (Ring 0) — no user-space process can
  intercept or fake it
  → Real Windows logon screen always appears
  → Fake logon screen attack impossible
```

For US government / DoD systems, the legal banner must include specific language per DISA STIG. For enterprise environments, consult legal team for jurisdiction-appropriate language.
