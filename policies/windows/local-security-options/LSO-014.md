# LSO-014 — Miscellaneous Critical Security Options

**ID:** LSO-014  
**Category:** Local Security Options / Miscellaneous  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Settings

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Audit: Shut down system immediately if unable to log
                      │    security audits: Disabled (workstations) / context-specific (DC)
                      ├─ Recovery console: Allow automatic administrative logon: Disabled
                      ├─ Shutdown: Allow system to be shut down without having
                      │    to log on: Disabled
                      └─ Microsoft network server: Disconnect clients when logon
                           hours expire: Enabled
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Lsa
  CrashOnAuditFail = 0   ; 0=don't crash, 1=BSOD if audit log full (DCs only)

HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System
  ShutdownWithoutLogon = 0   ; 0=require logon to shutdown (servers)

HKLM\SYSTEM\CurrentControlSet\Services\LanManServer\Parameters
  EnableForcedLogOff = 1     ; Disconnect clients when logon hours expire
```

## Each Setting Explained

| Setting | Value | Why |
|---|---|---|
| **CrashOnAuditFail** | 0 on workstations | 1 = BSOD if Security log fills → DoS risk. Use on sensitive DCs only |
| **ShutdownWithoutLogon** | 0 | Prevents physical attacker rebooting to recovery console without credentials |
| **EnableForcedLogOff** | 1 | Enforces logon hour restrictions — accounts don’t remain active after hours expire |

## CrashOnAuditFail Context

```
CrashOnAuditFail = 1 on Domain Controllers:
  Pro: Ensures every security event is logged (no silent failures)
  Con: If Security log fills without SIEM forwarding → DC goes down
  → Use ONLY if WEF is deployed and Security log forwarded in real-time
  → Set EventLog max size to 1GB+ before enabling

CrashOnAuditFail = 0 on workstations:
  Pro: Workstation outage avoided
  Con: Security events may be missed if log not forwarded
  → Acceptable on endpoints with WEF/SIEM forwarding
```
