# LSO-002 — NTLM Authentication Restrictions

**ID:** LSO-002  
**Category:** Local Security Options / NTLM  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Network security: LAN Manager authentication level:
                      │    Send NTLMv2 response only. Refuse LM & NTLM
                      ├─ Network security: Minimum session security for NTLM SSP based
                      │    (including secure RPC) clients:
                      │    Require NTLMv2 session security, Require 128-bit encryption
                      └─ Network security: Minimum session security for NTLM SSP based
                           (including secure RPC) servers:
                           Require NTLMv2 session security, Require 128-bit encryption
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Lsa
  LmCompatibilityLevel = 5
  ; 0=LM+NTLM, 1=LM+NTLM(NTLMv2 if negotiated), 2=NTLM only,
  ; 3=NTLMv2 only, 4=NTLMv2 only+refuse LM, 5=NTLMv2 only+refuse LM+NTLM

HKLM\SYSTEM\CurrentControlSet\Control\Lsa\MSV1_0
  NTLMMinClientSec = 537395200   ; NTLMv2 + 128-bit
  NTLMMinServerSec = 537395200   ; NTLMv2 + 128-bit
  ; 0x20000000 = NTLMv2 session security
  ; 0x20000000 = 128-bit encryption
  ; Combined: 537395200
```

## NTLM Version Comparison

| Version | Hash Algorithm | Crack Resistance | Status |
|---|---|---|---|
| **LM** | DES-based (56-bit) | 🔴 Breaks in seconds | Obsolete 1987 |
| **NTLMv1** | MD4 | 🔴 Breaks in minutes | Vulnerable |
| **NTLMv2** | HMAC-MD5 + challenge | 🟡 Resistant (no capture) | Required minimum |

## Attack Scenarios Blocked

```
Without this policy (LM/NTLMv1 allowed):
  1. Attacker sets up Responder on network
  2. Captures LM/NTLMv1 challenge-response
  3. Cracks hash offline in seconds/minutes
  4. Full credentials obtained

With LmCompatibilityLevel = 5:
  → LM hash: never sent
  → NTLMv1: refused
  → Only NTLMv2 sent → harder to crack without rainbow tables
  → Responder captures NTLMv2 but cracking is much harder

Note: Kerberos (not NTLM) should be primary auth — see ACC-008, ACC-012
```
