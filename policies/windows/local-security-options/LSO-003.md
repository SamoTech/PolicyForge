# LSO-003 — Disable LAN Manager Hash Storage

**ID:** LSO-003  
**Category:** Local Security Options / Credential Storage  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      └─ Network security: Do not store LAN Manager hash value
                           on next password change: Enabled
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Lsa
  NoLMHash = 1   ; 1=do not store LM hash, 0=store LM hash
```

## PowerShell

```powershell
# Check current setting
Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name NoLMHash

# Enable (do not store LM hash)
Set-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name NoLMHash -Value 1

# Note: Takes effect on NEXT password change
# Force all users to change password to purge existing LM hashes:
# (Domain) Set-ADUser -Identity * -ChangePasswordAtLogon $true
```

## Why LM Hash Is Catastrophic

```
LM Hash algorithm flaws:
  1. Password converted to UPPERCASE  → reduces keyspace by 50%
  2. Password padded to 14 chars with nulls
  3. Split into TWO 7-char chunks and hashed INDEPENDENTLY
  4. Each 7-char chunk: 26^7 uppercase = ~8 billion combinations

Cracking LM hash:
  hashcat -m 3000 lm_hash.txt  (LM hash type)
  RTX 4090: ~100 billion guesses/second on LM
  8 billion / 100 billion = 0.08 seconds per half
  14-char password → TWO 0.08-second cracks = 0.16 seconds total

  LM hash cracking: ALL passwords ≤ 14 chars in < 1 second

NTLM hash comparison:
  hashcat -m 1000 ntlm_hash.txt
  RTX 4090: ~500 million/second for NTLM (200x slower)
  12-char complex NTLM: ~years
```

## Important Note

This policy only prevents **new** LM hashes from being stored. Existing LM hashes in the SAM/NTDS.dit remain until users change their passwords. After enabling this policy, force a password change for all accounts to purge legacy LM hashes.
