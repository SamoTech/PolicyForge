# LSO-013 — Audit Policy: Force Subcategory Settings

**ID:** LSO-013  
**Category:** Local Security Options / Audit  
**Risk Level:** 🟠 High  
**OS:** Windows Vista+, Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      └─ Audit: Force audit policy subcategory settings (Windows Vista
                           or later) to override audit policy category settings: Enabled
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Lsa
  SCENoApplyLegacyAuditPolicy = 1   ; 1=use subcategory settings, 0=use legacy category settings
```

## Description

Windows has two audit policy systems:
1. **Legacy category-based** (9 categories, coarse-grained)
2. **Advanced subcategory-based** (56 subcategories, fine-grained)

Without `SCENoApplyLegacyAuditPolicy = 1`, the legacy category settings can **override** the advanced subcategory settings configured in AUDIT-001 through AUDIT-014. This policy ensures that the fine-grained audit policies (configured via `auditpol.exe`) take precedence and cannot be accidentally overwritten by a coarse GPO setting.

```
Without this policy:
  Advanced subcategory audit policies set via auditpol.exe
  → Overridden by legacy "Audit logon events: Success, Failure"
  → Fine-grained audit settings lost
  → SIEM loses critical event categories

With SCENoApplyLegacyAuditPolicy = 1:
  Legacy category settings ignored
  Subcategory settings from auditpol.exe / GPO always apply
  → All 56 subcategories work as configured
```
