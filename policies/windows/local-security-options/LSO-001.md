# LSO-001 — UAC: Admin Approval Mode & Elevation Behavior

**ID:** LSO-001  
**Category:** Local Security Options / UAC  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ UAC: Admin Approval Mode for the Built-in Administrator account: Enabled
                      ├─ UAC: Behavior of the elevation prompt for admins in Admin Approval Mode:
                      │    Prompt for credentials on the secure desktop
                      ├─ UAC: Behavior of the elevation prompt for standard users:
                      │    Automatically deny elevation requests
                      ├─ UAC: Detect application installations and prompt for elevation: Enabled
                      ├─ UAC: Only elevate UIAccess applications installed in secure locations: Enabled
                      ├─ UAC: Run all administrators in Admin Approval Mode: Enabled
                      ├─ UAC: Switch to the secure desktop when prompting for elevation: Enabled
                      └─ UAC: Virtualize file and registry write failures to per-user locations: Enabled
```

## Registry

```
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System
  EnableLUA                          = 1   ; UAC enabled
  FilterAdministratorToken           = 1   ; Admin Approval Mode for RID-500
  ConsentPromptBehaviorAdmin         = 1   ; 0=no prompt, 1=creds secure desktop, 2=consent secure desktop
                                           ; 3=creds, 4=consent, 5=consent apps only
  ConsentPromptBehaviorUser          = 0   ; 0=auto deny, 1=creds secure desktop, 3=prompt
  EnableInstallerDetection           = 1
  ValidateAdminCodeSignatures        = 0   ; 1=only signed apps can elevate (strict)
  EnableUIADesktopToggle             = 0
  PromptOnSecureDesktop              = 1   ; Forces elevation prompt on isolated secure desktop
  EnableVirtualization               = 1
```

## PowerShell

```powershell
# Check UAC configuration
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" |
    Select EnableLUA, FilterAdministratorToken, ConsentPromptBehaviorAdmin,
           ConsentPromptBehaviorUser, PromptOnSecureDesktop
```

## Why Secure Desktop Matters

```
UAC prompt WITHOUT secure desktop (PromptOnSecureDesktop = 0):
  → Prompt appears on normal desktop
  → Any running process can:
     - Simulate button clicks on the prompt
     - Inject input to dismiss/accept prompt
     - Read screen to detect prompt appearance
  → Malware can auto-elevate by clicking UAC prompt

UAC prompt WITH secure desktop (PromptOnSecureDesktop = 1):
  → Desktop is frozen for all other processes
  → Only winlogon.exe (SYSTEM) can render the prompt
  → No other process can interact with it
  → Malware cannot auto-click the prompt
```

## ConsentPromptBehaviorAdmin Values

| Value | Behavior | Security | Recommended |
|---|---|---|---|
| 0 | Elevate without prompting | 🔴 None | Never |
| 1 | Credentials on secure desktop | 🟢 Highest | ✅ Yes (CIS) |
| 2 | Consent on secure desktop | 🔵 High | Acceptable |
| 3 | Credentials (no secure desktop) | 🟡 Medium | No |
| 4 | Consent (no secure desktop) | 🟡 Medium | No |
| 5 | Consent for non-Windows (default) | 🟡 Medium | Minimum |
