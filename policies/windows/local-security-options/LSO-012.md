# LSO-012 — Network Security: LDAP Client Signing

**ID:** LSO-012  
**Category:** Local Security Options / LDAP Security  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Advisory ADV190023

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Network security: LDAP client signing requirements:
                      │    Negotiate signing (minimum) → Require signing (preferred)
                      └─ Domain controller: LDAP server signing requirements:
                           Require signing
```

## Registry

```
; Client
HKLM\SYSTEM\CurrentControlSet\Services\ldap
  LDAPClientIntegrity = 2   ; 0=none, 1=negotiate, 2=require

; Server (Domain Controller)
HKLM\SYSTEM\CurrentControlSet\Services\NTDS\Parameters
  LDAPServerIntegrity = 2   ; 0=none, 1=negotiate, 2=require
```

## LDAP Relay Attack Context

```
Without LDAP signing (CVE-2017-8563 / LDAP relay):
  1. Attacker triggers NTLM authentication (PetitPotam, PrinterBug)
  2. Relays authentication to LDAP (port 389) on DC
  3. Authenticates as machine account with DC replication rights
  4. Requests DCSync (domain replication)
  5. Dumps all domain hashes

With LDAP signing required:
  Step 2 → DC requires signed LDAP
  Attacker cannot forge valid LDAP signing
  → LDAP relay fails
  → DCSync blocked

Related: Use LDAPS (port 636) + LDAP channel binding for full protection
```
