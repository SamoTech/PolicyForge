# LSO-010 — Shutdown: Clear Virtual Memory Pagefile

**ID:** LSO-010  
**Category:** Local Security Options / Memory Security  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      └─ Shutdown: Clear virtual memory pagefile: Enabled
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management
  ClearPageFileAtShutdown = 1   ; 1=clear pagefile on shutdown, 0=don't clear
```

## Description

The Windows pagefile (pagefile.sys) contains swapped memory pages from all running processes — which can include plaintext passwords, encryption keys, decrypted documents, browser session tokens, and other sensitive data that was in RAM and swapped to disk. When the system shuts down normally, pagefile.sys remains on disk with all this data intact.

```
Pagefile forensics attack:
  1. Attacker physically accesses powered-off machine
  2. Removes drive (BitLocker bypassed or not enabled)
  3. Mounts drive on forensics workstation
  4. Runs strings / grep against pagefile.sys
  5. Finds: passwords, keys, session tokens, email content

With ClearPageFileAtShutdown = 1:
  → Pagefile zeroed on graceful shutdown
  → Cold boot / disk extraction finds empty pagefile

Note: Does NOT protect against suspend/hibernate — see hiberfil.sys
Note: Adds ~1-2 minutes to shutdown time on large pagefiles
```
