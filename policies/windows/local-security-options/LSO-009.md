# LSO-009 — Restrict Access to Floppy/CD-ROM for Locally Logged-On Users

**ID:** LSO-009  
**Category:** Local Security Options / Device Access  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Devices: Restrict CD-ROM access to locally logged-on user only: Enabled
                      ├─ Devices: Allowed to format and eject removable media: Administrators
                      └─ Devices: Prevent users from installing printer drivers: Enabled
```

## Registry

```
HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
  AllocateCDRoms   = "1"   ; CD-ROM accessible only to local user (string value)

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
  AllocateFloppies = "1"   ; Floppy accessible only to local user

HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Printers
  DisableWebPnPDownload = 1  ; Prevent printer driver installs via web
  NoAddPrinter = 1           ; Prevent non-admins from adding printers
```

## Description

When a user logs on locally, removable media (CD/DVD/USB) should only be accessible to that user — not accessible to network users simultaneously. This prevents a network-connected attacker from accessing removable media inserted by a local user. Blocking non-admin printer driver installation prevents a common privilege escalation vector (print spooler / driver abuse).

## Print Spooler Context

```
Printer driver installation privilege escalation:
  Non-admin installs malicious printer driver
  → Driver runs as SYSTEM (print spooler context)
  → Code execution as SYSTEM
  → CVE-2021-1675 / CVE-2021-34527 (PrintNightmare) exploited this path

Prevention: Administrators only can install printer drivers
```
