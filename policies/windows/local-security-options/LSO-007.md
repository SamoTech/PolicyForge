# LSO-007 — Disable Anonymous Enumeration (Network Access)

**ID:** LSO-007  
**Category:** Local Security Options / Anonymous Access  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Network access: Do not allow anonymous enumeration of SAM accounts: Enabled
                      ├─ Network access: Do not allow anonymous enumeration of SAM accounts and shares: Enabled
                      ├─ Network access: Let Everyone permissions apply to anonymous users: Disabled
                      ├─ Network access: Restrict anonymous access to Named Pipes and Shares: Enabled
                      └─ Network access: Shares that can be accessed anonymously: (blank)
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Lsa
  RestrictAnonymous              = 1   ; Restrict anonymous SAM/share access
  RestrictAnonymousSAM           = 1   ; No anonymous SAM queries
  EveryoneIncludesAnonymous      = 0   ; Everyone group excludes anonymous

HKLM\SYSTEM\CurrentControlSet\Services\LanManServer\Parameters
  NullSessionShares              = ""  ; No anonymous shares
  NullSessionPipes               = ""  ; No anonymous named pipes
  RestrictNullSessAccess         = 1   ; Restrict null session access
```

## What Anonymous Enumeration Exposes

```
With anonymous enumeration enabled (rpcclient / enum4linux):
  rpcclient -U "" -N 192.168.1.100
  > enumdomusers
  [+] Administrator (500)
  [+] john.smith (1001)
  [+] svc_backup (1005)
  [+] svc_sql (1010)
  > netshareenum
  [+] ADMIN$, C$, Finance, HR, Backup

With LSO-007 applied:
  rpcclient -U "" -N 192.168.1.100
  > enumdomusers → NT_STATUS_ACCESS_DENIED
  > netshareenum → NT_STATUS_ACCESS_DENIED
```
