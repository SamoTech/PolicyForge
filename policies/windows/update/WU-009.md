# WU-009 — Configure Update Deadlines (Force Install + Restart)

**ID:** WU-009  
**Category:** Windows Update / Compliance / Enforcement  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10 1903+, Windows 11  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Windows Update
                 └─ Manage end user experience
                      └─ Specify deadlines for automatic updates and restarts
                           ├─ Enabled
                           ├─ Quality update deadline: 7 days
                           ├─ Feature update deadline: 14 days
                           ├─ Grace period: 2 days
                           └─ Don't auto-restart until deadline: Enabled
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate
  ConfigureDeadlineForQualityUpdates     = 7    (days until forced install)
  ConfigureDeadlineForFeatureUpdates     = 14
  ConfigureDeadlineGracePeriod           = 2    (days user can postpone restart)
  ConfigureDeadlineNoAutoReboot          = 1    (wait for deadline before restart)
```

## PowerShell

```powershell
# Check deadline configuration
Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" | 
    Select ConfigureDeadlineForQualityUpdates, ConfigureDeadlineForFeatureUpdates, ConfigureDeadlineGracePeriod

# Check update pending restart status
$session = New-Object -ComObject Microsoft.Update.Session
$searcher = $session.CreateUpdateSearcher()
$result = $searcher.Search("IsInstalled=0")
Write-Host "Pending updates: $($result.Updates.Count)"

# Check restart pending registry keys
$pendingReboot = @(
    "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\RebootPending",
    "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\RebootRequired"
)
$pendingReboot | ForEach-Object { Test-Path $_ }
```

## Deadline Behavior Timeline

```
Day 0:   Update becomes available
Day 0-7: Update downloads, installs during active hours if possible
Day 7:   DEADLINE — forced install begins if not already done
Day 7-9: Grace period — user can postpone restart up to 2 more days
Day 9:   FORCED RESTART — machine reboots even if user is active
         (User gets warnings: 15 min, 5 min, 1 min before restart)
```

## Description

Deadline policy ensures that updates are installed within a maximum of 9 days (7-day deadline + 2-day grace period). Without deadlines, machines with WU-001's "no reboot with logged-on users" setting may never reboot if a user is always logged in. Deadlines provide the safety net: after the deadline, the machine will restart regardless, ensuring 100% patch compliance across the fleet even for users who avoid rebooting.

## Impact

- ✅ Guarantees 100% patch installation within 9 days
- ✅ Users get warnings before forced restart
- ✅ Complements WU-001 (no auto-reboot) with a backstop deadline
- ⚠️ Users will experience forced restarts after 9 days — communicate this
- ℹ️ Combine with Active Hours (WU-005) to keep deadline reboots to non-work hours
