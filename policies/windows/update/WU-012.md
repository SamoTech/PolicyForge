# WU-012 — Windows Update Compliance Audit Script

**ID:** WU-012  
**Category:** Windows Update / Compliance / Audit  
**Risk Level:** 🟢 Low  
**OS:** Windows 10+, Windows 11  
**Source:** PolicyForge Automation

---

## Description

Comprehensive PowerShell script to audit Windows Update compliance state across a single machine or fleet. Reports missing updates, last install dates, WSUS configuration, reboot pending status, and patch age. Use as a standalone script or integrate into your RMM/SCCM monitoring.

## Single Machine Audit

```powershell
#Requires -RunAsAdministrator
<#
.SYNOPSIS
    PolicyForge WU-012 - Windows Update Compliance Audit
.DESCRIPTION
    Audits Windows Update configuration, compliance, and patch age.
    Outputs a structured compliance report.
#>

function Get-WUComplianceReport {
    $report = [PSCustomObject]@{
        ComputerName        = $env:COMPUTERNAME
        OSVersion           = (Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion").DisplayVersion
        LastSuccessfulInstall = $null
        PatchAgeInDays      = $null
        PendingReboot       = $false
        WSUSConfigured      = $false
        WSUSServer          = "(direct)"
        AutoUpdateEnabled   = $false
        MissingUpdates      = 0
        CriticalMissing     = 0
        UpdatePolicy        = @{}
        ComplianceStatus    = "UNKNOWN"
    }

    # Last installed update
    $lastHotfix = Get-HotFix | Sort-Object InstalledOn -Descending |
                  Where-Object { $_.InstalledOn -ne $null } | Select-Object -First 1
    if ($lastHotfix) {
        $report.LastSuccessfulInstall = $lastHotfix.InstalledOn
        $report.PatchAgeInDays = (Get-Date) - $lastHotfix.InstalledOn | Select-Object -ExpandProperty Days
    }

    # Pending reboot check
    $rebootKeys = @(
        "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\RebootPending",
        "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\RebootRequired",
        "HKLM:\SYSTEM\CurrentControlSet\Control\Session Manager\PendingFileRenameOperations"
    )
    $report.PendingReboot = $rebootKeys | Where-Object { Test-Path $_ } | Measure-Object | Select-Object -ExpandProperty Count
    $report.PendingReboot = $report.PendingReboot -gt 0

    # WSUS configuration
    $wuPolicy = Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" -ErrorAction SilentlyContinue
    if ($wuPolicy.WUServer) {
        $report.WSUSConfigured = $true
        $report.WSUSServer = $wuPolicy.WUServer
    }

    # AU settings
    $auPolicy = Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" -ErrorAction SilentlyContinue
    $report.AutoUpdateEnabled = ($auPolicy.NoAutoUpdate -eq 0) -or ($null -eq $auPolicy.NoAutoUpdate)

    # Deferral settings
    $report.UpdatePolicy = @{
        QualityDeferDays  = $wuPolicy.DeferQualityUpdatesPeriodInDays
        FeatureDeferDays  = $wuPolicy.DeferFeatureUpdatesPeriodInDays
        TargetVersion     = $wuPolicy.TargetReleaseVersionInfo
        LocalMergeEnabled = -not ($wuPolicy.ExcludeWUDriversInQualityUpdate -eq 1)
    }

    # Compliance status
    $report.ComplianceStatus = switch ($true) {
        ($report.PatchAgeInDays -gt 30)  { "NON_COMPLIANT - Patches > 30 days old" }
        ($report.PatchAgeInDays -gt 14)  { "WARNING - Patches > 14 days old" }
        ($report.PendingReboot)           { "WARNING - Pending reboot" }
        ($not $report.AutoUpdateEnabled)  { "NON_COMPLIANT - Auto updates disabled" }
        default                           { "COMPLIANT" }
    }

    return $report
}

# Run audit
$compliance = Get-WUComplianceReport
$compliance | Format-List

# Export to JSON for SIEM/RMM ingestion
$compliance | ConvertTo-Json -Depth 3 | Out-File "C:\PolicyForge\WU-Compliance-$env:COMPUTERNAME.json"
```

## Fleet Audit (via Invoke-Command)

```powershell
# Run compliance check across all domain computers
$computers = Get-ADComputer -Filter { Enabled -eq $true } | Select-Object -ExpandProperty Name

$results = Invoke-Command -ComputerName $computers -ScriptBlock {
    $lastHotfix = Get-HotFix | Sort-Object InstalledOn -Descending |
                  Where-Object { $_.InstalledOn -ne $null } | Select-Object -First 1
    [PSCustomObject]@{
        Computer    = $env:COMPUTERNAME
        LastPatch   = $lastHotfix.InstalledOn
        PatchAge    = if ($lastHotfix) { ((Get-Date) - $lastHotfix.InstalledOn).Days } else { 999 }
        NeedsReboot = Test-Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\RebootRequired"
    }
} -ErrorAction SilentlyContinue

# Show non-compliant machines (patch age > 14 days)
$results | Where-Object { $_.PatchAge -gt 14 } |
    Sort-Object PatchAge -Descending |
    Format-Table Computer, LastPatch, PatchAge, NeedsReboot -AutoSize
```

## Compliance Thresholds

| Patch Age | Status | Action |
|---|---|---|
| 0-7 days | ✅ Compliant | No action |
| 7-14 days | ⚠️ Warning | Monitor |
| 14-30 days | 🟠 Alert | Investigate + remediate |
| > 30 days | 🔴 Non-Compliant | Immediate remediation |
| > 60 days | 💀 Critical | Escalate — possible WU break |
