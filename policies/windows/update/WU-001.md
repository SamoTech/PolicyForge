# WU-001 — Configure Automatic Updates (No Auto-Install Without Approval)

**ID:** WU-001  
**Category:** Windows Update / Baseline  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Windows Update
                 └─ Manage end user experience
                      └─ Configure Automatic Updates
                           ├─ Enabled
                           ├─ Configure automatic updating: 4 - Auto download and schedule the install
                           ├─ Scheduled install day: Every day
                           └─ Scheduled install time: 03:00
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU
  NoAutoUpdate          = 0      (0=enabled, 1=disabled)
  AUOptions             = 4      (2=notify, 3=auto-download notify install, 4=auto-download schedule, 5=allow local admin)
  ScheduledInstallDay   = 0      (0=every day, 1=Sunday ... 7=Saturday)
  ScheduledInstallTime  = 3      (3 = 3:00 AM)
  NoAutoRebootWithLoggedOnUsers = 1
```

## PowerShell

```powershell
# Check current Windows Update settings
Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU"

# Verify last update check
(New-Object -ComObject Microsoft.Update.AutoUpdate).Results

# Force update check
wuauclt /detectnow
# Or on Windows 10+:
Get-WindowsUpdate  # Requires PSWindowsUpdate module

# Check pending updates
Get-HotFix | Sort-Object InstalledOn -Descending | Select-Object -First 10
```

## Description

Configures Windows Update to automatically download updates but schedule installation for off-hours (3:00 AM daily). This balances security (patches applied promptly) with operational stability (no surprise reboots during business hours). Combined with the no-reboot-with-logged-on-users setting, updates install overnight without disrupting active sessions.

## AUOptions Values Reference

| Value | Behavior | Recommended For |
|---|---|---|
| 2 | Notify before download | Strict change control environments |
| 3 | Auto-download, notify install | Semi-managed endpoints |
| **4** | **Auto-download, schedule install** | **Standard enterprise (recommended)** |
| 5 | Allow local admin to configure | Not recommended (inconsistent) |

## Impact

- ✅ Updates download automatically, reducing patch lag
- ✅ Installation at 3 AM avoids business-hours disruption
- ✅ No forced reboots while users are logged in
- ⚠️ Requires endpoints to be powered on at scheduled time
- ℹ️ Use Wake-on-LAN or Active Hours to complement this
