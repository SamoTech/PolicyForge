# WU-002 — Configure WSUS Server (Centralized Update Management)

**ID:** WU-002  
**Category:** Windows Update / WSUS  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11, Windows Server 2016+  
**Source:** Microsoft WSUS Documentation · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Windows Update
                 └─ Manage updates offered from Windows Server Update Service
                      ├─ Specify intranet Microsoft update service location
                      │    ├─ Enabled
                      │    ├─ Set the intranet update service for detecting updates: http://wsus-server:8530
                      │    └─ Set the intranet statistics server: http://wsus-server:8530
                      └─ Do not connect to any Windows Update Internet locations
                           └─ Enabled
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate
  WUServer          = "http://wsus-server.domain.local:8530"
  WUStatusServer    = "http://wsus-server.domain.local:8530"
  DisableWindowsUpdateAccess = 0

HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU
  UseWUServer = 1    (1=use WSUS, 0=use Microsoft Update)
```

## PowerShell

```powershell
# Verify WSUS configuration on client
$wuServer = (Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate").WUServer
Write-Host "WSUS Server: $wuServer"

# Check if client is communicating with WSUS
$au = Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU"
Write-Host "Using WSUS: $($au.UseWUServer)"

# Force WSUS registration
wuauclt /resetauthorization /detectnow

# Check Windows Update service status
Get-Service wuauserv, bits, cryptsvc | Select Name, Status, StartType

# WSUS server-side: approve critical + security updates automatically
# (Run on WSUS server)
Import-Module UpdateServices
$wsus = Get-WsusServer
$subscription = $wsus.GetSubscription()
$approvalRules = $wsus.GetInstallApprovalRules()
```

## WSUS Port Reference

| Protocol | Port | Use |
|---|---|---|
| HTTP | 8530 | Default WSUS (no SSL) |
| HTTPS | 8531 | Recommended for production |
| HTTP | 80 | Legacy IIS-based WSUS |
| HTTPS | 443 | Legacy IIS-based WSUS |

## Description

Directs Windows Update clients to an internal WSUS server instead of Microsoft's public update servers. WSUS provides centralized control: administrators approve updates before they deploy, see compliance status across all endpoints, and prevent endpoints from going directly to the internet for updates. This is essential for air-gapped networks and environments with strict change control requirements.

## Impact

- ✅ Centralized patch approval and scheduling
- ✅ Bandwidth savings (updates cached internally)
- ✅ Full visibility into update compliance across fleet
- ✅ Prevents unauthorized feature updates
- ⚠️ WSUS server becomes a single point of failure — plan for HA
- ⚠️ WSUS requires maintenance (disk space, database cleanup)
