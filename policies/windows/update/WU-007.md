# WU-007 — Configure Delivery Optimization (Bandwidth Control)

**ID:** WU-007  
**Category:** Windows Update / Delivery Optimization / Network  
**Risk Level:** 🟢 Low  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft Documentation · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Delivery Optimization
                 ├─ Download Mode: 1 (LAN only) or 100 (Bypass/WSUS only)
                 ├─ Max download bandwidth: 50% during business hours
                 ├─ Max upload bandwidth: 5 GB/month
                 └─ Group ID: [your org GUID]
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows\DeliveryOptimization
  DODownloadMode              = 1      (0=off, 1=LAN, 2=Group, 3=Internet, 99=simple, 100=bypass)
  DOMaxDownloadBandwidth      = 0      (0=unlimited, value in KB/s)
  DOPercentageMaxDownloadBandwidth = 50  (50% during active hours)
  DOMaxUploadBandwidth        = 5120   (5GB/month upload cap)
  DOGroupId                   = "{your-org-guid}"
  DOSetHoursToLimitBackground = 1
  DOActiveHoursStart          = 8
  DOActiveHoursEnd            = 18
```

## PowerShell

```powershell
# Check current Delivery Optimization settings
Get-DeliveryOptimizationStatus

# Check DO mode
(Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\DeliveryOptimization").DODownloadMode

# View DO statistics
Get-DeliveryOptimizationPerfSnap

# View DO perf log
Get-DeliveryOptimizationLog | Select -Last 50
```

## Download Mode Reference

| Mode | Value | Description | Use Case |
|---|---|---|---|
| Off | 0 | No P2P, direct internet | Highly restricted networks |
| **LAN** | **1** | **P2P within LAN only** | **Standard enterprise (recommended)** |
| Group | 2 | P2P within AD group | Multi-site organizations |
| Internet | 3 | P2P over internet | Home users, SMB |
| Simple | 99 | Direct download only | Air-gapped environments |
| Bypass | 100 | Use WSUS/SCCM only | Full WSUS control |

## Description

Delivery Optimization (DO) enables peer-to-peer update sharing between machines on the same LAN, reducing internet bandwidth consumption. Mode 1 (LAN) is optimal for enterprise: machines share updates with each other locally, dramatically reducing redundant downloads when 500 machines all need the same 500 MB update. The bandwidth throttle during active hours (50%) prevents update downloads from saturating business-hours network capacity.

## Impact

- ✅ Significant WAN bandwidth savings (LAN P2P sharing)
- ✅ Faster updates on LAN (local cache hit instead of internet download)
- ✅ Business-hours bandwidth protection (50% cap)
- ⚠️ Mode 3 (Internet) should never be used in enterprise — leaks update metadata
- ℹ️ Use Mode 100 (Bypass) if fully managed by WSUS/SCCM
