# WU-003 — Defer Feature Updates (Prevent Premature OS Upgrades)

**ID:** WU-003  
**Category:** Windows Update / Feature Update Management  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ Windows Update
                 └─ Manage updates offered from Windows Update
                      ├─ Select the target Feature Update version
                      │    ├─ Enabled
                      │    ├─ Which Windows product version to receive: Windows 11
                      │    └─ Target Version: 23H2   (pin to specific version)
                      └─ Specify deadlines for automatic updates and restarts
                           ├─ Quality updates deadline: 7 days
                           └─ Feature updates deadline: 30 days
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate
  TargetReleaseVersion      = 1
  TargetReleaseVersionInfo  = "23H2"    (set to target version string)
  ProductVersion            = "Windows 11"

; Deferral periods (Windows Update for Business)
  DeferFeatureUpdatesPeriodInDays  = 30
  DeferQualityUpdatesPeriodInDays  = 7
```

## PowerShell

```powershell
# Check current Windows version
[System.Environment]::OSVersion.Version
(Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion").DisplayVersion

# Check feature update deferral settings
Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" |
    Select TargetReleaseVersion, TargetReleaseVersionInfo, DeferFeatureUpdatesPeriodInDays

# Check update history
Get-WinEvent -LogName System | Where-Object { $_.Id -eq 19 } | Select -First 10
# Event ID 19 = Windows Update installed successfully
```

## Feature Update Version Reference (Windows 11)

| Version | Name | Release | EOL (Enterprise) | Status |
|---|---|---|---|---|
| 21H2 | Original | Oct 2021 | Oct 2024 | ⛔ EOL |
| 22H2 | 2022 Update | Sep 2022 | Oct 2025 | ⚠️ Near EOL |
| 23H2 | 2023 Update | Oct 2023 | Nov 2026 | ✅ Supported |
| 24H2 | 2024 Update | Oct 2024 | Oct 2027 | ✅ Latest |

## Description

Pins endpoints to a specific Windows feature update version, preventing automatic upgrades to the next major version. Feature updates (e.g., 22H2 → 23H2) can break applications, change security configurations, and require significant testing before enterprise deployment. Deferring for 30 days allows Microsoft to identify and patch post-release issues before your fleet receives it. Use TargetReleaseVersionInfo to pin to a specific version until your IT team completes compatibility testing.

## Impact

- ✅ Prevents surprise major OS upgrades breaking LOB applications
- ✅ 30-day buffer to catch post-release regressions
- ✅ Controlled, tested rollout of feature updates
- ⚠️ Must update the target version before it reaches EOL
- ℹ️ Quality/security updates still apply on the 7-day schedule
