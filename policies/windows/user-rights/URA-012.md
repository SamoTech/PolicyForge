# URA-012 — Restrict "Manage Auditing and Security Log" (SeSecurityPrivilege)

**ID:** URA-012  
**Category:** User Rights Assignment / Audit Integrity  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Manage auditing and security log: Administrators
```

## secedit Template

```ini
[Privilege Rights]
SeSecurityPrivilege = *S-1-5-32-544  ; Administrators only
```

## Description

`SeSecurityPrivilege` controls who can:
- Configure audit policies (SACL)
- Clear the Security event log (Event 1102)
- Read the Security event log
- View/modify the audit settings of any object

This privilege is the "attacker's eraser" — if an attacker gains SeSecurityPrivilege, they can clear all evidence of their activity from the Security log. It should be restricted to Administrators only, and clearing of the Security log (Event 1102) should be treated as a **P1 security incident** regardless of who performs it (see AUDIT-009).

## Critical Monitoring Rule

```
If Event ID 1102 (Security Log Cleared) appears:
  → Immediate P1 incident response
  → Assume: attacker had SeSecurityPrivilege
  → Assume: all prior security events destroyed
  → Check WEF/SIEM for forwarded copies
  → Forensic: check Windows event forwarding gaps
```
