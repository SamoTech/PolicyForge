# URA-003 — Restrict "Create Token Object" (SeCreateTokenPrivilege)

**ID:** URA-003  
**Category:** User Rights Assignment / Token Security  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Create a token object: (No accounts)
```

## secedit Template

```ini
[Privilege Rights]
SeCreateTokenPrivilege =   ; Empty = no accounts
```

## Description

`SeCreateTokenPrivilege` allows a process to call `ZwCreateToken()` directly, creating arbitrary access tokens. This means an attacker with this privilege can craft a token claiming to be any user, any group, any privilege level — essentially creating fake administrator credentials from thin air. The Windows kernel uses this internally; no user-space account should ever hold it.
