# URA-004 — Restrict Network Logon Rights

**ID:** URA-004  
**Category:** User Rights Assignment / Network Access  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      ├─ Access this computer from the network:
                      │    Administrators, Remote Desktop Users
                      └─ Deny access to this computer from the network:
                           Guests, Local account (S-1-5-113),
                           Local account and member of Administrators group (S-1-5-114)
```

## secedit Template

```ini
[Privilege Rights]
SeNetworkLogonRight         = *S-1-5-32-544,*S-1-5-32-555  ; Admins + RDP Users
SeDenyNetworkLogonRight     = *S-1-5-32-546,*S-1-5-113,*S-1-5-114
; S-1-5-32-546 = Guests
; S-1-5-113    = Local account (blocks all local accounts from network logon)
; S-1-5-114    = Local admin accounts (prevents local admin lateral movement)
```

## PowerShell

```powershell
# Check who can access this computer from the network
Secedit /export /cfg C:\PolicyForge\rights.inf /quiet
Select-String "SeNetworkLogonRight|SeDenyNetworkLogonRight" C:\PolicyForge\rights.inf
```

## Why Block Local Accounts from Network Logon?

The critical security win here is denying `S-1-5-113` (all local accounts) and `S-1-5-114` (local admin accounts) from network logon. This prevents **Pass-the-Hash lateral movement**:

```
Attack Scenario (without this policy):
  1. Attacker compromises workstation-A
  2. Dumps local admin hash from SAM
  3. Uses hash to authenticate to workstation-B (same local admin password)
  4. Spreads across entire fleet

With this policy:
  Step 3 → BLOCKED: local accounts cannot authenticate via network
  Result: Lateral movement stopped at workstation-A
```

This is the **single most effective policy against Pass-the-Hash lateral movement** in environments that haven't fully moved to LAPS or unique local admin passwords.
