# URA-011 — Restrict Anonymous SID/Name Translation & Share Access

**ID:** URA-011  
**Category:** User Rights Assignment / Anonymous Access  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Access this computer from the network: (remove Everyone, Anonymous)

  └─ Security Options
       ├─ Network access: Do not allow anonymous enumeration of SAM accounts: Enabled
       ├─ Network access: Do not allow anonymous enumeration of SAM accounts and shares: Enabled
       ├─ Network access: Let Everyone permissions apply to anonymous users: Disabled
       └─ Network access: Restrict anonymous access to Named Pipes and Shares: Enabled
```

## Registry

```
HKLM\SYSTEM\CurrentControlSet\Control\Lsa
  RestrictAnonymous        = 1   (restrict anonymous enumeration)
  RestrictAnonymousSAM     = 1   (prevent anonymous SAM enumeration)
  EveryoneIncludesAnonymous = 0  (Everyone group excludes anonymous)
```

## Description

Anonymous access to Windows systems was historically enabled for backward compatibility. Without these restrictions, unauthenticated users on the network can:

- Enumerate all local user accounts (SAM database query)
- List network shares
- List named pipes
- Enumerate group memberships
- Query LSA policy information

This information is gold for reconnaissance — attackers use tools like `enum4linux`, `rpcclient`, and `net use \\target\IPC$` for anonymous enumeration before launching targeted credential attacks.

## Reconnaissance Prevention

```
With anonymous access enabled (vulnerable):
  enum4linux -a 192.168.1.100
  → [+] Users: Administrator, john.smith, svc_backup...
  → [+] Shares: ADMIN$, C$, Finance, HR
  → [+] Groups: Administrators (Administrator, john.smith)

With URA-011 applied:
  enum4linux -a 192.168.1.100
  → [-] Failed to enumerate users (access denied)
  → [-] Failed to list shares (access denied)
```
