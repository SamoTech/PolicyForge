# URA-013 — Restrict "Impersonate a Client After Authentication" (SeImpersonatePrivilege)

**ID:** URA-013  
**Category:** User Rights Assignment / Privilege Escalation  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Impersonate a client after authentication:
                           Administrators, LOCAL SERVICE, NETWORK SERVICE, SERVICE
```

## secedit Template

```ini
[Privilege Rights]
SeImpersonatePrivilege = *S-1-5-32-544,*S-1-5-19,*S-1-5-20,*S-1-5-6
; S-1-5-32-544 = Administrators
; S-1-5-19     = LOCAL SERVICE
; S-1-5-20     = NETWORK SERVICE
; S-1-5-6      = Service
```

## Description

`SeImpersonatePrivilege` is abused by **Potato attacks** — a family of privilege escalation exploits (RottenPotato, JuicyPotato, PrintSpoofer, GodPotato, SweetPotato) that use this privilege to escalate from a low-privileged service account to SYSTEM.

## Potato Attack Chain

```
Prerequisite: Service account with SeImpersonatePrivilege
  (Web server running as IIS AppPool, SQL service account, etc.)

Attack:
  1. Attacker achieves RCE as IIS/SQL service account
  2. Runs JuicyPotato.exe / PrintSpoofer.exe
  3. Abuses SeImpersonatePrivilege to impersonate SYSTEM token
  4. Achieves SYSTEM shell

Prevention:
  → Remove SeImpersonatePrivilege from IIS AppPool accounts
  → Use gMSA for service accounts (automatic, managed)
  → Apply this URA to restrict who can impersonate
```

## Important Note

IIS application pools, SQL Server service accounts, and other service identities often have this privilege by default. Audit service accounts regularly:

```powershell
# Find all local accounts with SeImpersonatePrivilege
Secedit /export /cfg C:\PolicyForge\ura_audit.inf /quiet
Select-String "SeImpersonatePrivilege" C:\PolicyForge\ura_audit.inf
```
