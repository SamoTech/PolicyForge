# URA-008 — Restrict "Take Ownership" Privilege (SeTakeOwnershipPrivilege)

**ID:** URA-008  
**Category:** User Rights Assignment / ACL Bypass  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Take ownership of files or other objects: Administrators
```

## secedit Template

```ini
[Privilege Rights]
SeTakeOwnershipPrivilege = *S-1-5-32-544  ; Administrators only
```

## Description

`SeTakeOwnershipPrivilege` allows any process to take ownership of any securable object — files, directories, registry keys, processes, threads, tokens — regardless of existing ACLs. Once ownership is taken, the account can change the DACL to grant itself full control, effectively bypassing all file system and registry security.

An attacker with this privilege can:
- Take ownership of any file (including protected system files)
- Modify ACLs to grant themselves read/write access
- Read/overwrite any registry key including SAM
- Compromise any service or process
