# URA-009 — Restrict "Load and Unload Device Drivers" (SeLoadDriverPrivilege)

**ID:** URA-009  
**Category:** User Rights Assignment / Kernel Protection  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Load and unload device drivers: Administrators
```

## secedit Template

```ini
[Privilege Rights]
SeLoadDriverPrivilege = *S-1-5-32-544  ; Administrators only
```

## Description

`SeLoadDriverPrivilege` allows loading arbitrary kernel-mode drivers into the Windows kernel. A malicious driver runs at Ring 0 (kernel level), giving complete control of the system — all security software, EDR, AV can be disabled or bypassed from kernel level. This is the access level used by rootkits and advanced persistent threats (APTs).

## Kernel Attack Path

```
Attacker with SeLoadDriverPrivilege:
  1. Load vulnerable legitimate driver (CVE-XXXX-YYYY)
  2. Exploit vulnerability to execute code at kernel level
  3. Disable EDR/AV driver from kernel space
  4. Achieve undetected persistence

Real Examples:
  - BYOVD (Bring Your Own Vulnerable Driver) attacks
  - BlackByte ransomware used RTCore64.sys
  - Lazarus Group used Dell DBUtil driver
```
