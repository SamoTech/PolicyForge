# URA-002 — Restrict "Act as Part of the Operating System" (SeTcbPrivilege)

**ID:** URA-002  
**Category:** User Rights Assignment / OS-Level Privilege  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ User Rights Assignment
                      └─ Act as part of the operating system: (No accounts)
```

## secedit Template

```ini
[Privilege Rights]
SeTcbPrivilege =   ; Empty = no accounts
```

## Description

`SeTcbPrivilege` is one of the most powerful privileges in Windows — it grants an account the ability to act as a trusted part of the OS itself. A process with this privilege can:

- Call `LsaLogonUser()` to create any type of logon session
- Generate arbitrary security tokens without authentication
- Bypass virtually all access controls
- Impersonate any user including SYSTEM

This privilege should be assigned to **no accounts** in a hardened environment. If any account holds it, that account can compromise the entire system regardless of other security controls.
