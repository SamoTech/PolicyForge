# ACC-008 — Kerberos Policy Hardening

**ID:** ACC-008  
**Category:** Account Policies / Kerberos Policy  
**Risk Level:** 🔴 Critical  
**OS:** Windows Server 2016+ (Domain Controllers)  
**Source:** CIS Benchmark Windows Server 2022 · Microsoft Security Baseline · DISA STIG

---

## Policy Path

```
Default Domain Policy → Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Account Policies
                 └─ Kerberos Policy
                      ├─ Enforce user logon restrictions: Enabled
                      ├─ Maximum lifetime for service ticket: 600 minutes (10 hours)
                      ├─ Maximum lifetime for user ticket: 10 hours
                      ├─ Maximum lifetime for user ticket renewal: 7 days
                      └─ Maximum tolerance for computer clock synchronization: 5 minutes
```

## secedit (Default Domain Policy)

```ini
[Kerberos Policy]
MaxTicketAge       = 10    ; hours — user TGT lifetime
MaxServiceAge      = 600   ; minutes — service ticket lifetime
MaxRenewAge        = 7     ; days — TGT renewal window
MaxClockSkew       = 5     ; minutes — max clock drift allowed
TicketValidateClient = 1   ; enforce user logon restrictions
```

## Kerberoasting Mitigation Context

```
Kerberoasting attack:
  1. Any domain user requests service tickets for SPNs
  2. Service ticket is encrypted with service account's NTLM hash
  3. Attacker takes ticket offline and cracks it
  4. If cracked: service account password exposed

Kerberos policy impact:
  MaxServiceAge = 600 min → ticket expires in 10 hours
  (Offline cracking has no time constraint — ticket lifetime
   does not stop cracking, but limits ticket replay window)

Real Kerberoasting mitigations (see ACC-012):
  → Use AES-256 encryption only (disable RC4)
  → Managed Service Accounts (gMSA) with 128-char passwords
  → Monitor Event 4769 (Kerberos service ticket requests)
```

## Clock Skew — Critical for Kerberos

```
MaxClockSkew = 5 minutes is enforced by Kerberos protocol:
  If client clock differs from DC by > 5 minutes:
  → ALL Kerberos authentication fails
  → Users cannot log on

Ensure NTP is configured on all domain members:
  w32tm /query /status
  w32tm /resync /force
```
