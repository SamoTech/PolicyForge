# ACC-002 — Maximum Password Age

**ID:** ACC-002  
**Category:** Account Policies / Password Policy  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · NIST 800-63B

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Account Policies
                 └─ Password Policy
                      └─ Maximum password age: 365 days (or per org policy)
```

## secedit

```ini
[System Access]
MaximumPasswordAge = 365
```

## The Modern Debate: Expiration vs. No Expiration

| Stance | Recommendation | Rationale |
|---|---|---|
| **CIS Benchmark** | ≤ 365 days | Limits exposure window if credential is compromised |
| **NIST 800-63B (2017+)** | No mandatory rotation | Forced rotation leads to predictable patterns (Password1! → Password2!) |
| **Microsoft** | No expiration if MFA is enforced | MFA renders password age less critical |
| **DISA STIG** | 60 days | Government/military high-security environments |

## PolicyForge Recommendation

```
If MFA is enforced for all accounts:
  → MaximumPasswordAge = 0 (never expire) + require MFA everywhere
  → Monitor for breached credentials via HIBP/Azure AD Identity Protection

If MFA is NOT universally enforced:
  → MaximumPasswordAge = 90 days for privileged accounts
  → MaximumPasswordAge = 365 days for standard users
  → Pair with Fine-Grained Password Policy (ACC-010)
```
