# ACC-003 — Minimum Password Age

**ID:** ACC-003  
**Category:** Account Policies / Password Policy  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Account Policies
                 └─ Password Policy
                      └─ Minimum password age: 1 day
```

## secedit

```ini
[System Access]
MinimumPasswordAge = 1
```

## Description

Setting a minimum password age of **1 day** prevents users from immediately changing their password multiple times to exhaust the password history and return to their original password. This policy works in tandem with ACC-001 (Password History = 24).

```
Without minimum age (vulnerability):
  User forced to change password
  → Changes 24 times in rapid succession
  → Returns to original password
  → Password history bypassed completely

With minimum age = 1 day:
  → Must wait 1 day between each change
  → 24 changes = 24 days minimum
  → Password cycling attack becomes impractical
```
