# ACC-011 — Privileged Account Hardening (Admin Account Controls)

**ID:** ACC-011  
**Category:** Account Policies / Privileged Account Management  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft PAW guidance

---

## Policy Settings

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      ├─ Accounts: Rename administrator account: [custom name]
                      ├─ Accounts: Rename guest account: [custom name]
                      ├─ Accounts: Guest account status: Disabled
                      └─ Accounts: Administrator account status: Disabled
                           (use domain admin accounts instead)
```

## secedit

```ini
[System Access]
EnableAdminAccount = 0     ; Disable built-in Administrator (RID 500)
EnableGuestAccount = 0     ; Disable built-in Guest (RID 501)

[Registry Values]
MACHINE\Software\Microsoft\Windows NT\CurrentVersion\Winlogon\AllocateDASD = 4,"0"
```

## PowerShell — Admin Account Hardening

```powershell
# Disable built-in Administrator (SID ending in -500)
Disable-LocalUser -Name "Administrator"

# Rename built-in Administrator (security through obscurity layer)
Rename-LocalUser -Name "Administrator" -NewName "IT_SVC_MGMT"

# Disable Guest
Disable-LocalUser -Name "Guest"

# Ensure renamed admin has a strong, unique password (pre-LAPS)
$secPwd = ConvertTo-SecureString "$(New-Guid)-$(New-Guid)" -AsPlainText -Force
Set-LocalUser -Name "IT_SVC_MGMT" -Password $secPwd

# Verify no additional unexpected local admin accounts
Get-LocalGroupMember -Group "Administrators" |
    Where-Object { $_.PrincipalSource -ne 'ActiveDirectory' }
```

## Admin Account Attack Vectors

| Vector | Risk | Mitigation |
|---|---|---|
| Default name "Administrator" | Attacker knows exact username | Rename account |
| Enabled + no LAPS | Same password fleet-wide | Disable + use LAPS |
| RID 500 always exists | Cannot be deleted | Rename + disable |
| Guest enabled | Anonymous/low-priv access | Disable always |
| Unexpected local admins | Shadow admin persistence | Audit regularly |

## Privileged Access Workstation (PAW) Context

For Tier 0 admin accounts (Domain Admins, Enterprise Admins), Microsoft recommends **dedicated PAW workstations** — separate physical or virtualized machines used exclusively for administrative tasks, never for email or web browsing. This policy is the baseline; PAW is the full solution.
