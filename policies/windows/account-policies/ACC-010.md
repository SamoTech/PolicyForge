# ACC-010 — Fine-Grained Password Policies (PSOs)

**ID:** ACC-010  
**Category:** Account Policies / Fine-Grained Password Policy  
**Risk Level:** 🟠 High  
**OS:** Windows Server 2008 R2+ (Active Directory Domain)  
**Source:** Microsoft AD DS Documentation · CIS Benchmark

---

## Description

Fine-Grained Password Policies (Password Settings Objects / PSOs) allow **different password policies for different user groups** within the same AD domain — overcoming the single-domain-policy limitation. Use this to enforce much stricter policies for privileged accounts.

## PowerShell — Create PSOs

```powershell
# Import Active Directory module
Import-Module ActiveDirectory

# === PSO 1: Privileged Accounts (Admins, Service Accounts) ===
New-ADFineGrainedPasswordPolicy `
    -Name "PSO-PrivilegedAccounts" `
    -DisplayName "Privileged Account Password Policy" `
    -Precedence 10 `
    -MinPasswordLength 20 `
    -PasswordHistoryCount 24 `
    -MaxPasswordAge "60.00:00:00" `
    -MinPasswordAge "1.00:00:00" `
    -ComplexityEnabled $true `
    -ReversibleEncryptionEnabled $false `
    -LockoutThreshold 3 `
    -LockoutDuration "00:30:00" `
    -LockoutObservationWindow "00:30:00"

# === PSO 2: Service Accounts ===
New-ADFineGrainedPasswordPolicy `
    -Name "PSO-ServiceAccounts" `
    -DisplayName "Service Account Password Policy" `
    -Precedence 20 `
    -MinPasswordLength 30 `
    -PasswordHistoryCount 24 `
    -MaxPasswordAge "365.00:00:00" `
    -ComplexityEnabled $true `
    -ReversibleEncryptionEnabled $false `
    -LockoutThreshold 3 `
    -LockoutDuration "-00:00:01"   `   # Never auto-unlock (admin only)
    -LockoutObservationWindow "00:30:00"

# === Apply PSOs to security groups ===
Add-ADFineGrainedPasswordPolicySubject -Identity "PSO-PrivilegedAccounts" `
    -Subjects "Domain Admins", "Enterprise Admins", "IT-Administrators"

Add-ADFineGrainedPasswordPolicySubject -Identity "PSO-ServiceAccounts" `
    -Subjects "Service-Accounts"

# === Verify applied PSO ===
Get-ADUserResultantPasswordPolicy -Identity "john.admin"
```

## PSO Precedence Logic

```
If a user is member of multiple groups with PSOs:
  → The PSO with the LOWEST Precedence number wins
  → Example: Precedence 10 beats Precedence 20
  → Always assign lowest numbers to most restrictive policies

If both a user object and group PSO apply:
  → User object PSO always wins regardless of precedence number
```

## PolicyForge Recommended PSO Tiers

| Tier | Applies To | Min Length | Max Age | Lockout |
|---|---|---|---|---|
| **PSO-Privileged (P=10)** | Domain Admins, Tier 0 | 20 chars | 60 days | 3 attempts |
| **PSO-ServiceAccts (P=20)** | All service accounts | 30 chars | 365 days | 3, admin-unlock |
| **PSO-Standard (P=30)** | Standard users | 14 chars | 365 days | 5 attempts |
| **Default Domain Policy** | Fallback | 14 chars | 365 days | 5 attempts |
