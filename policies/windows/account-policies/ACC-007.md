# ACC-007 — Account Lockout Duration & Reset Counter

**ID:** ACC-007  
**Category:** Account Policies / Account Lockout  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Account Policies
                 └─ Account Lockout Policy
                      ├─ Account lockout duration: 15 minutes
                      └─ Reset account lockout counter after: 15 minutes
```

## secedit

```ini
[System Access]
LockoutDuration = 15
ResetLockoutCount = 15
```

## Description

These two settings work together with ACC-006 (lockout threshold):

- **LockoutDuration = 15**: Locked account automatically unlocks after 15 minutes. Setting to `0` requires manual administrator unlock — stronger but generates more helpdesk tickets.
- **ResetLockoutCount = 15**: The bad attempt counter resets to 0 after 15 minutes of no failed attempts. This prevents attackers from making 4 attempts every hour indefinitely without ever triggering lockout.

## Slow Brute Force Attack (Mitigated)

```
Without ResetLockoutCount restriction:
  Attacker makes 4 attempts, waits, makes 4 more, waits...
  Never triggers 5-attempt lockout
  → Slow password spray over days/weeks

With ResetLockoutCount = 15 minutes:
  Counter resets only after 15 minutes of no attempts
  But combined with ACC-006 (threshold = 5):
  → Still blocks rapid-fire attempts
  → Password spray detection via SIEM (Event 4625 volume)
```

## Admin Unlock Consideration

```
LockoutDuration = 0 (admin unlock only):
  Pro: Attacker cannot wait out a lockout and resume attack
  Con: User calls helpdesk after any lockout
  → Use for privileged accounts only (via Fine-Grained Policy, ACC-010)

LockoutDuration = 15 (auto-unlock):
  Pro: Reduces helpdesk load
  Con: Attacker can wait 15 min and try 5 more
  → Acceptable for standard users with SIEM monitoring
```
