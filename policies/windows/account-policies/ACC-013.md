# ACC-013 — Monitor & Alert on Account Policy Events

**ID:** ACC-013  
**Category:** Account Policies / Account Monitoring  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** NIST 800-53 · Microsoft MDE · CIS Benchmark

---

## Critical Account Event IDs

| Event ID | Log | Meaning | Priority |
|---|---|---|---|
| **4740** | Security | Account locked out | 🔴 P1 — possible brute force |
| **4625** | Security | Failed logon attempt | 🔴 P1 if volume spike |
| **4648** | Security | Logon using explicit credentials | 🟠 P2 — lateral movement indicator |
| **4720** | Security | User account created | 🔴 P1 — unauthorized account? |
| **4722** | Security | User account enabled | 🟠 P2 |
| **4723** | Security | Password change attempt | 🟡 P3 |
| **4724** | Security | Password reset attempt | 🟠 P2 |
| **4725** | Security | User account disabled | 🟠 P2 |
| **4726** | Security | User account deleted | 🔴 P1 |
| **4738** | Security | User account changed | 🟠 P2 |
| **4756** | Security | Member added to universal group | 🔴 P1 if Admins group |
| **4732** | Security | Member added to local group | 🔴 P1 if Admins group |

## PowerShell — Account Monitoring Queries

```powershell
# === Lockout storm detection ===
# More than 10 lockouts in 5 minutes = potential password spray
$lockouts = Get-WinEvent -FilterHashtable @{
    LogName = 'Security'; Id = 4740
    StartTime = (Get-Date).AddMinutes(-5)
} -ErrorAction SilentlyContinue

if ($lockouts.Count -gt 10) {
    Write-Warning "ALERT: $($lockouts.Count) lockouts in last 5 minutes!"
    Write-Warning "Possible password spray attack. Check source IPs."
    $lockouts | Select TimeCreated,
        @{N='User';E={$_.Properties[0].Value}},
        @{N='CallerMachine';E={$_.Properties[1].Value}}
}

# === Failed logon spike (brute force indicator) ===
$failures = Get-WinEvent -FilterHashtable @{
    LogName = 'Security'; Id = 4625
    StartTime = (Get-Date).AddMinutes(-5)
} -ErrorAction SilentlyContinue

if ($failures.Count -gt 50) {
    Write-Warning "ALERT: $($failures.Count) failed logons in 5 minutes!"
}

# === Unexpected admin group additions ===
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4732} |
    Where-Object { $_.Properties[2].Value -match 'Administrators' } |
    Select TimeCreated,
        @{N='AddedUser';E={$_.Properties[0].Value}},
        @{N='Group';E={$_.Properties[2].Value}},
        @{N='Actor';E={$_.Properties[6].Value}}

# === New account creation audit ===
Get-WinEvent -FilterHashtable @{
    LogName='Security'; Id=4720
    StartTime=(Get-Date).AddDays(-1)
} | Select TimeCreated,
    @{N='NewAccount';E={$_.Properties[0].Value}},
    @{N='CreatedBy';E={$_.Properties[4].Value}}
```

## Password Spray Detection Pattern

```
Password Spray signature in event logs:
  Many Event 4625 with:
    - Different usernames
    - Same source IP
    - Same time window
    - Same failure reason (0xC000006A = wrong password)

Brute Force (single account) signature:
    - Same username
    - Many 4625 events
    - Followed by 4740 (lockout)

KQL (Azure Sentinel / Defender):
  SecurityEvent
  | where EventID == 4625
  | summarize count() by IpAddress, bin(TimeGenerated, 5m)
  | where count_ > 50
```
