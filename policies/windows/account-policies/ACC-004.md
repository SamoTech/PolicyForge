# ACC-004 — Minimum Password Length

**ID:** ACC-004  
**Category:** Account Policies / Password Policy  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · NIST 800-63B · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Account Policies
                 └─ Password Policy
                      ├─ Minimum password length: 14
                      └─ Minimum Password Length Audit: Enabled (Windows 10 2004+)
```

## secedit

```ini
[System Access]
MinimumPasswordLength = 14
RelaxMinimumPasswordLengthLimits = 1  ; Allows lengths > 14 (up to 128)
```

## Password Length vs. Cracking Time

| Length | Charset (upper+lower+digit+symbol) | Crack Time (offline, GPU) |
|---|---|---|
| 8 chars | 94^8 = 6.1 trillion | **~2 hours** (RTX 4090) |
| 10 chars | 94^10 = 53 quadrillion | ~1 week |
| 12 chars | 94^12 = 4.7 × 10²³ | ~92 years |
| **14 chars** | 94^14 = 3.9 × 10²⁷ | **~810,000 years** |
| 16 chars | 94^16 | ~72 billion years |
| 20 chars (passphrase) | 94^20 | Heat death of universe |

> For NTLM hashes (still common): 8-char passwords crack in seconds with Hashcat on modern GPUs. Length is the single most impactful password security control.

## Passphrase Strategy (NIST 800-63B)

NIST recommends **passphrases** over complex short passwords:

```
Weak (complex, short):  P@ssw0rd1!    (8 chars, common pattern)
Strong (simple, long):  correct-horse-battery-staple  (28 chars, memorable)
Enterprise target:      minimum 14 chars, no complexity requirement → use passphrase
```

## Windows 10 2004+ Enhancement

`RelaxMinimumPasswordLengthLimits = 1` removes the historical 14-char GPO cap, allowing minimum lengths up to 128 characters. Enable this on modern systems.
