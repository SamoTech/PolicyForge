# ACC-012 — Disable RC4 / Enforce AES Kerberos Encryption

**ID:** ACC-012  
**Category:** Account Policies / Kerberos Encryption  
**Risk Level:** 🔴 Critical  
**OS:** Windows Server 2008 R2+, Windows 10+  
**Source:** Microsoft Security Advisory · CIS Benchmark · NIST

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ System
            └─ KDC (on Domain Controllers)
                 └─ KDC support for claims, compound auth, and Kerberos armoring: Enabled

Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Local Policies
                 └─ Security Options
                      └─ Network security: Configure encryption types allowed for Kerberos:
                           ✅ AES128_HMAC_SHA1
                           ✅ AES256_HMAC_SHA1
                           ✅ Future encryption types
                           ❌ DES_CBC_CRC (uncheck)
                           ❌ DES_CBC_MD5 (uncheck)
                           ❌ RC4_HMAC_MD5 (uncheck — carefully, test first)
```

## Registry

```
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Kerberos\Parameters
  SupportedEncryptionTypes = 0x7FFFFFF8  ; AES128 + AES256 + future only
  ; Decimal: 2147483640

; Individual flags:
; 0x4   = DES_CBC_CRC     (disabled)
; 0x8   = DES_CBC_MD5     (disabled)
; 0x10  = RC4_HMAC_MD5    (disabled)
; 0x8   = AES128_HMAC_SHA1 (enabled)
; 0x10  = AES256_HMAC_SHA1 (enabled)
```

## PowerShell

```powershell
# Check current Kerberos encryption settings
Get-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Kerberos\Parameters" `
    -Name SupportedEncryptionTypes -ErrorAction SilentlyContinue

# Check AD user accounts that still use RC4
Get-ADUser -Filter * -Properties msDS-SupportedEncryptionTypes |
    Where-Object { $_.'msDS-SupportedEncryptionTypes' -band 4 } |   # RC4 bit
    Select SamAccountName, 'msDS-SupportedEncryptionTypes'

# Set user account to require AES only
Set-ADUser -Identity "svc_sql" -KerberosEncryptionType AES256, AES128
```

## Kerberoasting + RC4 Connection

```
Kerberoasting with RC4 (default, insecure):
  Service ticket encrypted with RC4 (NTLM hash)
  → Offline crack with Hashcat: medium-complexity 8-char = minutes
  → RC4 is fast to crack: billions of guesses/second

Kerberoasting with AES256 (enforced):
  Service ticket encrypted with AES-256
  → Offline crack: 10,000x slower than RC4
  → Strong password + AES = effectively uncrackable
  → gMSA (128-char password) + AES = impossible to crack
```

## ⚠️ RC4 Removal Warning

Disabling RC4 can break:
- Legacy applications using RC4 for Kerberos
- Older NAS/printer devices that only support RC4
- Some cross-forest trusts with older DCs

**Always test in audit/lab environment first. Remove RC4 in phases.**
