# ACC-009 — Windows LAPS Deployment & Hardening

**ID:** ACC-009  
**Category:** Account Policies / Local Admin Password Management  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 20H2+, Windows 11, Server 2019+  
**Source:** Microsoft LAPS Documentation · CIS Benchmark · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ System
            └─ LAPS
                 ├─ Configure password backup directory: Azure AD / Active Directory
                 ├─ Password Settings:
                 │    ├─ Password complexity: Large letters + small letters + numbers + specials (4)
                 │    ├─ Password length: 15 (minimum)
                 │    └─ Password age (days): 30
                 ├─ Post-authentication actions: Reset password + sign out managed account (3)
                 └─ Enable password encryption: Enabled
```

## Registry

```
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\LAPS\Config
  BackupDirectory         = 2    (1=AD, 2=AAD/Entra ID)
  PasswordComplexity      = 4    (4=all char types)
  PasswordLength          = 15
  PasswordExpirationProtectionEnabled = 1
  ADPasswordEncryptionEnabled = 1
  PostAuthenticationResetDelay = 24  ; hours
  PostAuthenticationActions   = 3    ; reset pwd + sign out
```

## PowerShell — Deploy Windows LAPS

```powershell
# Check if Windows LAPS is available (built-in since Win11 22H2)
Get-WindowsCapability -Online -Name 'LAPS*'

# Update AD schema for Windows LAPS (run as Enterprise Admin)
Update-LapsADSchema

# Grant computers permission to write their own LAPS attribute
Set-LapsADComputerSelfPermission -Identity "OU=Workstations,DC=corp,DC=com"

# Configure LAPS via GPO or Intune (registry approach):
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\LAPS\Config" `
    -Name "BackupDirectory" -Value 2
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\LAPS\Config" `
    -Name "PasswordLength" -Value 15
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\LAPS\Config" `
    -Name "PasswordComplexity" -Value 4

# Retrieve LAPS password (helpdesk/admin use)
Get-LapsADPassword -Identity "WORKSTATION01" -AsPlainText

# Force LAPS password rotation
Invoke-LapsPolicyProcessing
Reset-LapsPassword
```

## Why LAPS Is Critical

```
Without LAPS (common enterprise problem):
  All 500 workstations share the same local admin password
  → Attacker compromises ONE workstation
  → Dumps local admin hash from SAM
  → Pass-the-Hash to ALL 500 workstations
  → Complete lateral movement across fleet

With LAPS deployed:
  Every workstation has a unique, random 15-char local admin password
  → Compromising one workstation's local admin hash
  → Works ONLY on that one workstation
  → Lateral movement completely stopped
  → Pairs perfectly with URA-004 (deny network logon for local accounts)
```

## LAPS vs Legacy LAPS

| Feature | Legacy LAPS | Windows LAPS (Built-in) |
|---|---|---|
| **Availability** | Separate MSI install | Built-in Windows 10 20H2+ |
| **Password storage** | AD only (clear text attr) | AD (encrypted) or Entra ID |
| **Password encryption** | ❌ No | ✅ AES-256 |
| **Post-auth rotation** | ❌ No | ✅ Auto-rotates after use |
| **Passphrase support** | ❌ No | ✅ Yes |
| **Entra ID support** | ❌ No | ✅ Yes |

## Post-Authentication Actions

Setting `PostAuthenticationActions = 3` (reset + sign out) means the local admin password is automatically rotated after it's used for authentication. This prevents reuse of retrieved passwords — critical for helpdesk workflows.
