# FW-010 — Configure IPSec / Connection Security Rules

**ID:** FW-010  
**Category:** Windows Firewall / IPSec / Connection Security  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11, Windows Server 2016+  
**Source:** Microsoft IPSec Documentation · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ Connection Security Rules
                      └─ New Rule > Isolation
```

## PowerShell

```powershell
# Create a Connection Security Rule requiring Kerberos authentication
# for all connections between domain members
New-NetIPsecRule -DisplayName "Domain Member Isolation" `
                 -InboundSecurity Require `
                 -OutboundSecurity Request `
                 -Phase1AuthSet "ComputerKerberosSet" `
                 -Phase2AuthSet "UserKerberosSet" `
                 -InterfaceType Any

# Alternatively, require encryption for sensitive traffic (e.g., to SQL servers):
New-NetIPsecRule -DisplayName "Encrypt Traffic to SQL" `
                 -InboundSecurity Require `
                 -OutboundSecurity Require `
                 -Phase2AuthSet "UserKerberosSet" `
                 -RemoteAddress "10.0.2.0/24" `  # SQL subnet
                 -Protocol TCP `
                 -LocalPort Any `
                 -RemotePort 1433

# Check active IPSec rules
Get-NetIPsecRule | Select DisplayName, Enabled, InboundSecurity, OutboundSecurity

# Check active IPSec SAs (Security Associations)
Get-NetIPsecMainModeSA
Get-NetIPsecQuickModeSA
```

## Description

IPSec Connection Security Rules enforce mutual authentication and optional encryption between domain members. Domain Isolation uses IPSec to ensure only domain-authenticated computers can communicate with each other — preventing rogue or compromised systems from talking to domain resources. This is a strong defense against lateral movement from non-domain or compromised systems that have valid credentials but no valid machine certificate.

## Use Cases

| Use Case | Rule Type | Config |
|---|---|---|
| Domain Isolation | Isolation | Require Kerberos from all domain members |
| Server Isolation | Isolation | Restrict server access to specific groups |
| Encrypted sensitive traffic | Custom | Require encryption to database/financial servers |
| Block non-domain systems | Isolation | Request auth — non-domain systems get blocked |

## Impact

- ✅ Only domain-authenticated machines can communicate
- ✅ Optional traffic encryption for sensitive server traffic
- ⚠️ Complex to configure and troubleshoot — test thoroughly
- ⚠️ Non-domain devices (vendors, guests) need exemption rules
