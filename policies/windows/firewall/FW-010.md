# FW-010 — Restrict WinRM & Remote Management Ports

**ID:** FW-010  
**Category:** Windows Firewall / Remote Management  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ Inbound Rules
                      ├─ WinRM (HTTP) TCP 5985: Block on Public, restrict on Domain
                      ├─ WinRM (HTTPS) TCP 5986: Allow Domain from mgmt IPs only
                      ├─ PowerShell Remoting: TCP 5985/5986 — management hosts only
                      └─ WMI: TCP 135 + dynamic RPC — management hosts only
```

## PowerShell

```powershell
# Disable WinRM HTTP (5985) — use HTTPS only
New-NetFirewallRule -DisplayName "Block WinRM HTTP Inbound" `
    -Direction Inbound -Protocol TCP -LocalPort 5985 `
    -Action Block -Profile Any -Enabled True

# Allow WinRM HTTPS from management subnet only
$mgmtSubnet = "10.0.100.0/24"  # Management VLAN
New-NetFirewallRule -DisplayName "Allow WinRM HTTPS Mgmt Only" `
    -Direction Inbound -Protocol TCP -LocalPort 5986 `
    -RemoteAddress $mgmtSubnet `
    -Action Allow -Profile Domain -Enabled True

# Block WinRM HTTPS from all other sources
New-NetFirewallRule -DisplayName "Block WinRM HTTPS Default" `
    -Direction Inbound -Protocol TCP -LocalPort 5986 `
    -Action Block -Profile Any -Enabled True

# Restrict RPC endpoint mapper (WMI)
New-NetFirewallRule -DisplayName "Allow RPC EpMap Mgmt Only" `
    -Direction Inbound -Protocol TCP -LocalPort 135 `
    -RemoteAddress $mgmtSubnet `
    -Action Allow -Profile Domain -Enabled True

# Check WinRM status
Get-WSManInstance winrm/config/listener -Enumerate
Get-Service WinRM | Select Name, Status, StartType
```

## WinRM Attack Context

```
WinRM without restrictions:
  Any authenticated user can run PowerShell commands remotely:
    Invoke-Command -ComputerName target -ScriptBlock {whoami}
  → Lateral movement via PS remoting
  → Invoke-Command, Enter-PSSession, WMI

With FW-010:
  WinRM only reachable from management VLAN
  Attacker on workstation VLAN cannot PS-remote to other workstations
  Management traffic isolated to management hosts
  → Lateral movement via WinRM significantly impeded

Pairs with:
  URA-003: Deny network logon for local accounts
  URA-010: Restrict WinRM access via user rights
```
