# FW-004 — Block Inbound LLMNR & NetBIOS (Poisoning Defense)

**ID:** FW-004  
**Category:** Windows Firewall / Lateral Movement Prevention  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark · SANS · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ Inbound Rules
                      ├─ Block: UDP 5355 (LLMNR) — All profiles
                      ├─ Block: UDP 137 (NetBIOS Name Service) — All profiles
                      ├─ Block: UDP 138 (NetBIOS Datagram) — All profiles
                      └─ Block: TCP 139 (NetBIOS Session) — All profiles
```

## PowerShell

```powershell
# Block LLMNR
New-NetFirewallRule -DisplayName "Block LLMNR Inbound" `
    -Direction Inbound -Protocol UDP -LocalPort 5355 `
    -Action Block -Profile Any -Enabled True

# Block NetBIOS Name Service
New-NetFirewallRule -DisplayName "Block NetBIOS-NS Inbound" `
    -Direction Inbound -Protocol UDP -LocalPort 137 `
    -Action Block -Profile Any -Enabled True

# Block NetBIOS Datagram
New-NetFirewallRule -DisplayName "Block NetBIOS-DGM Inbound" `
    -Direction Inbound -Protocol UDP -LocalPort 138 `
    -Action Block -Profile Any -Enabled True

# Block NetBIOS Session
New-NetFirewallRule -DisplayName "Block NetBIOS-SSN Inbound" `
    -Direction Inbound -Protocol TCP -LocalPort 139 `
    -Action Block -Profile Any -Enabled True

# Disable LLMNR via Group Policy (preferred)
# Computer Config → Admin Templates → DNS Client → Turn off multicast name resolution: Enabled
```

## LLMNR/NBT-NS Poisoning Attack Chain

```
LLMNR Poisoning (Responder):
  1. User types "\\filserver" (typo — server doesn't exist)
  2. DNS lookup fails
  3. Windows falls back to LLMNR broadcast (UDP 5355)
     "Who is filserver?"
  4. Responder on network responds: "I am filserver"
  5. Windows authenticates via NTLMv2 to attacker
  6. Responder captures NTLMv2 hash
  7. Crack offline or relay to other services

Firewall block (FW-004) + LLMNR disable (GPO):
  Step 3: LLMNR broadcast blocked by firewall
  Step 3 (alt): LLMNR disabled entirely via GPO
  → Responder never receives broadcast
  → No NTLMv2 capture possible

Pairs with:
  LSO-002: NTLMv2 only (harder to crack)
  LSO-004: SMB signing (relay blocked)
```

## Disable LLMNR via Registry

```
HKLM\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient
  EnableMulticast = 0   ; Disable LLMNR

HKLM\SYSTEM\CurrentControlSet\Services\NetBT\Parameters
  NodeType = 2          ; 1=B-node (broadcast only), 2=P-node (point-to-point, no broadcast)
                        ; 4=M-node, 8=H-node (default)
```
