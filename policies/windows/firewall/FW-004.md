# FW-004 — Block Inbound RDP (Port 3389) on Public Profile

**ID:** FW-004  
**Category:** Windows Firewall / Remote Access Control  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark · CISA Advisory

---

## Policy Path

```
Firewall > Inbound Rules
  └─ Block TCP/UDP 3389 on Public profile
  └─ Restrict TCP/UDP 3389 on Domain/Private to specific source IPs
```

## PowerShell

```powershell
# Block RDP inbound on Public profile
New-NetFirewallRule -DisplayName "Block RDP Inbound Public" `
                   -Direction Inbound -Protocol TCP -LocalPort 3389 `
                   -Action Block -Profile Public -Enabled True

# On Domain profile: allow only from jump server / IT subnet
New-NetFirewallRule -DisplayName "Allow RDP from IT Jump Server" `
                   -Direction Inbound -Protocol TCP -LocalPort 3389 `
                   -RemoteAddress "10.0.1.0/24" `
                   -Action Allow -Profile Domain -Enabled True

# Block RDP from all other sources on Domain profile
New-NetFirewallRule -DisplayName "Block RDP Other Sources" `
                   -Direction Inbound -Protocol TCP -LocalPort 3389 `
                   -Action Block -Profile Domain -Enabled True

# Check existing RDP rules
Get-NetFirewallRule | Where-Object { $_.DisplayName -like "*Remote Desktop*" -or $_.DisplayName -like "*RDP*" } |
    Select DisplayName, Direction, Action, Enabled
```

## Description

RDP (Remote Desktop Protocol) is one of the most attacked services on the internet. Exposed RDP on port 3389 is routinely targeted by ransomware operators for initial access. This policy blocks inbound RDP on the Public profile entirely, and on Domain/Private restricts it to known source IPs (jump servers, IT subnet). RDP brute force and credential stuffing campaigns specifically target TCP 3389 exposed to the internet or lateral network segments.

## RDP Hardening Checklist

- [ ] FW-004: Block inbound 3389 on Public profile
- [ ] Restrict Domain/Private profile to jump server IPs only
- [ ] Enable Network Level Authentication (NLA) — see CRED batch
- [ ] Change RDP port from 3389 (security through obscurity, reduces noise)
- [ ] Enable RDP-specific account lockout (5 failed attempts)
- [ ] Use RD Gateway instead of direct RDP exposure
