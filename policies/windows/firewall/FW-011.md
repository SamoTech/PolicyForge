# FW-011 — Authenticated Firewall Bypass (Admin Exceptions)

**ID:** FW-011  
**Category:** Windows Firewall / Authenticated Bypass  
**Risk Level:** 🟡 Medium  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** Microsoft Security Baseline

---

## Description

Authenticated firewall bypass rules allow specific users or computers (identified by AD security group) to bypass standard firewall rules. This is used for monitoring agents, security scanners, and administrative tools that need unrestricted network access to endpoints — without opening ports for all traffic.

## PowerShell

```powershell
# Create authenticated bypass rule for security scanner accounts
# Only computers in the "Security Scanners" group can bypass
$securityScannerGroup = "S-1-5-21-XXXXXX-XXXXXX-XXXXXX-1234"  # SID of scanner group

New-NetFirewallRule -DisplayName "PolicyForge: Authenticated Bypass for Scanners" `
    -Direction Inbound `
    -Action Allow `
    -Authentication Required `
    -RemoteMachine $securityScannerGroup `
    -Profile Domain `
    -Enabled True

# View all authenticated bypass rules
Get-NetFirewallRule | Where-Object { $_.Action -eq 'Allow' -and $_.Authentication -ne 'NotRequired' } |
    Select DisplayName, Direction, Action, Authentication

# Audit all firewall exceptions (any Allow rule)
Get-NetFirewallRule -Direction Inbound -Action Allow -Enabled True |
    Select DisplayName, Profile, RemoteAddress, LocalPort |
    Sort-Object DisplayName
```

## Use Cases

| Tool | Why Bypass Needed | Scope |
|---|---|---|
| Vulnerability scanners (Nessus, Qualys) | Needs to probe all ports | Scanner host only |
| SCCM/Endpoint Management | Needs WMI, Admin shares | SCCM server only |
| EDR agents | Outbound telemetry | Specific cloud IPs |
| Network monitoring (SNMP) | Inbound SNMP polls | Monitoring server only |

## Security Rules for Bypass Rules

```
✔ Always scope bypass rules to specific computer/user AD groups
✔ Use machine authentication (Kerberos IPsec) for computer-based bypass
✔ Document every bypass rule with owner, purpose, review date
✔ Review bypass rules quarterly — remove stale rules
✔ Log bypass rule matches — alert on unexpected source IPs using bypass
✗ Never create unauthenticated bypass rules (Action=Allow, no authentication)
✗ Never use RemoteAddress=Any for bypass rules
```
