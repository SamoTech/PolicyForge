# FW-011 — Disable Firewall Rule Merging from Local Store

**ID:** FW-011  
**Category:** Windows Firewall / Policy Integrity  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ [Profile] Properties > Settings > Customize
                      ├─ Apply local firewall rules: No
                      └─ Apply local connection security rules: No
```

## Registry

```
; Domain Profile (most important)
HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\DomainProfile
  AllowLocalPolicyMerge           = 0
  AllowLocalIPsecPolicyMerge      = 0
```

## PowerShell

```powershell
# Disable local rule merging for Domain profile
Set-NetFirewallProfile -Profile Domain `
                       -AllowLocalFirewallRules False `
                       -AllowLocalIPsecRules False

# Verify
Get-NetFirewallProfile -Profile Domain | Select AllowLocalFirewallRules, AllowLocalIPsecRules
```

## Description

By default, Windows merges locally-created firewall rules with GPO-pushed rules. This means a local administrator (or malware with admin rights) can create a local firewall rule that opens a port even if GPO intends it to be closed. Disabling local rule merging on the Domain profile ensures that only GPO-defined rules are enforced — local rules are completely ignored. This is critical for firewall policy integrity in domain environments.

## Impact

- ✅ GPO firewall policy cannot be overridden by local rules
- ✅ Malware with local admin rights cannot open firewall holes
- ⚠️ Local admins can no longer add firewall exceptions without GPO
- ⚠️ Applications that auto-create firewall rules on install will be blocked
- ℹ️ Test carefully — some software installers rely on local firewall rule creation
