# FW-001 — Windows Firewall: Enable All Profiles & Fail-Secure

**ID:** FW-001  
**Category:** Windows Firewall / Profile Enforcement  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 ├─ Domain Profile
                 │    ├─ Firewall state: On
                 │    ├─ Inbound connections: Block (default)
                 │    └─ Outbound connections: Allow (default)
                 ├─ Private Profile
                 │    ├─ Firewall state: On
                 │    ├─ Inbound connections: Block (default)
                 │    └─ Outbound connections: Allow (default)
                 └─ Public Profile
                      ├─ Firewall state: On
                      ├─ Inbound connections: Block (default)
                      └─ Outbound connections: Block (default)
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\DomainProfile
  EnableFirewall            = 1
  DefaultInboundAction      = 1   ; 0=allow, 1=block
  DefaultOutboundAction     = 0
  DisableNotifications      = 1   ; Suppress user firewall notifications (managed)

HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\PrivateProfile
  EnableFirewall            = 1
  DefaultInboundAction      = 1
  DefaultOutboundAction     = 0
  DisableNotifications      = 1

HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\PublicProfile
  EnableFirewall            = 1
  DefaultInboundAction      = 1
  DefaultOutboundAction     = 1   ; Block outbound on public networks
  DisableNotifications      = 0   ; Allow notifications on public (user awareness)
```

## PowerShell

```powershell
# Enable all profiles and set defaults
Set-NetFirewallProfile -Profile Domain,Private -Enabled True `
    -DefaultInboundAction Block -DefaultOutboundAction Allow

Set-NetFirewallProfile -Profile Public -Enabled True `
    -DefaultInboundAction Block -DefaultOutboundAction Block

# Verify
Get-NetFirewallProfile | Select Name, Enabled, DefaultInboundAction, DefaultOutboundAction
```

## Profile Behavior Explained

| Profile | Active When | Inbound | Outbound | Rationale |
|---|---|---|---|---|
| **Domain** | On corp AD network | Block | Allow | Trust corp apps; restrict inbound |
| **Private** | Home/trusted network | Block | Allow | Less strict than public |
| **Public** | Airport, café, hotel | Block | Block | Zero trust — no unsolicited traffic |

## Why Public Outbound Block Matters

```
Public network (airport WiFi) without outbound block:
  Malware on machine reaches C2 server freely
  Exfiltration over HTTP/HTTPS passes through
  No corporate proxy to inspect traffic

With Public outbound block (DefaultOutboundAction = 1):
  Only explicitly allowed outbound rules pass
  Requires explicit rule for browser, VPN client, etc.
  Unknowns blocked by default
  Malware C2 communication blocked unless whitelisted
```
