# FW-003 — Block Inbound SMB (Port 445) on Public Profile

**ID:** FW-003  
**Category:** Windows Firewall / Lateral Movement Prevention  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft Security Baseline · NSA Advisory

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall > Inbound Rules
                 └─ Block TCP 445 (SMB) — Public Profile
```

## PowerShell

```powershell
# Block SMB inbound on Public profile (already blocked by default if FW-001 is applied)
# Explicit block rule for defense-in-depth:
New-NetFirewallRule -DisplayName "Block SMB Inbound Public" `
                   -Direction Inbound `
                   -Protocol TCP `
                   -LocalPort 445 `
                   -Action Block `
                   -Profile Public `
                   -Enabled True `
                   -Description "Block inbound SMB on public networks"

# Also block on Domain/Private for endpoints (workstations don't need inbound SMB)
New-NetFirewallRule -DisplayName "Block SMB Inbound Workstation" `
                   -Direction Inbound `
                   -Protocol TCP `
                   -LocalPort 445 `
                   -Action Block `
                   -Profile Domain,Private `
                   -Enabled True `
                   -Description "Workstations don't need inbound SMB from peers"

# Verify
Get-NetFirewallRule | Where-Object { $_.DisplayName -like "*SMB*" } |
    Select DisplayName, Direction, Action, Enabled, Profile
```

## netsh Equivalent

```cmd
netsh advfirewall firewall add rule name="Block SMB Inbound Public" ^^
    dir=in action=block protocol=TCP localport=445 profile=public enable=yes
```

## Description

Blocks inbound SMB (TCP 445) connections. SMB is the primary protocol for Windows file sharing and the attack vector for EternalBlue (MS17-010/WannaCry), NotPetya, and most Windows lateral movement techniques. Workstations have no legitimate need to accept inbound SMB connections from other workstations. This rule should be applied on all profiles for non-server endpoints, and on Public profile for all devices.

## Ports Covered

| Port | Protocol | Service | Block On |
|---|---|---|---|
| 445 | TCP | SMB (direct) | All profiles (workstations) |
| 139 | TCP | SMB (NetBIOS) | All profiles (workstations) |
| 137-138 | UDP | NetBIOS Name Service | All profiles (workstations) |
