# FW-012 — Block Inbound mDNS / NetBIOS Name Resolution

**ID:** FW-012  
**Category:** Windows Firewall / Network Discovery / Privacy  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark · Security Research

---

## PowerShell

```powershell
# Block inbound mDNS (UDP 5353) on Public profile
New-NetFirewallRule -DisplayName "Block mDNS Inbound Public" `
                   -Direction Inbound -Protocol UDP -LocalPort 5353 `
                   -Action Block -Profile Public -Enabled True

# Block NetBIOS Name Service inbound on all profiles
New-NetFirewallRule -DisplayName "Block NetBIOS NS Inbound" `
                   -Direction Inbound -Protocol UDP -LocalPort 137 `
                   -Action Block -Profile Any -Enabled True

# Block NetBIOS Datagram inbound
New-NetFirewallRule -DisplayName "Block NetBIOS DG Inbound" `
                   -Direction Inbound -Protocol UDP -LocalPort 138 `
                   -Action Block -Profile Any -Enabled True

# Block NetBIOS Session inbound
New-NetFirewallRule -DisplayName "Block NetBIOS SS Inbound" `
                   -Direction Inbound -Protocol TCP -LocalPort 139 `
                   -Action Block -Profile Any -Enabled True

# Also: disable LLMNR via GPO
# Computer Config > Admin Templates > Network > DNS Client
# > Turn off multicast name resolution = Enabled
```

## Description

mDNS (UDP 5353) and NetBIOS (UDP 137/138, TCP 139) are legacy name resolution protocols that leak network information and are abused for MITM attacks. LLMNR and NetBIOS poisoning (Responder tool) is one of the most common internal network attacks — an attacker on the same network responds to broadcast name queries and captures NTLM hashes. Blocking inbound mDNS/NetBIOS combined with disabling LLMNR via Group Policy eliminates this attack vector.
