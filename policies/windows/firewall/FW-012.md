# FW-012 — GPO-Based Firewall Rule Deployment & Merge Policy

**ID:** FW-012  
**Category:** Windows Firewall / GPO Deployment  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ [Each Profile] → Settings
                      ├─ Apply local firewall rules: No
                      └─ Apply local connection security rules: No
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\DomainProfile
  AllowLocalPolicyMerge            = 0   ; Do not merge local rules
  AllowLocalIPsecPolicyMerge       = 0   ; Do not merge local IPsec rules

HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\PrivateProfile
  AllowLocalPolicyMerge            = 0
  AllowLocalIPsecPolicyMerge       = 0

HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\PublicProfile
  AllowLocalPolicyMerge            = 0
  AllowLocalIPsecPolicyMerge       = 0
```

## Description

By default, local firewall rules created by users, applications, or local admins are **merged** with GPO-delivered rules. This means a user or malware with local admin rights can create a `netsh advfirewall` rule that opens any port — bypassing GPO restrictions.

```
Local merge enabled (default):
  GPO: Block TCP 4444 inbound
  Local admin: netsh advfirewall firewall add rule name="C2" protocol=TCP
               localport=4444 action=allow dir=in
  Result: Port 4444 OPEN (local rule overrides GPO block)

Local merge disabled (AllowLocalPolicyMerge = 0):
  GPO: Block TCP 4444 inbound
  Local admin: adds same local rule
  Result: Port 4444 BLOCKED (local rules ignored, GPO only)
  → GPO is authoritative — cannot be bypassed by local changes
```

## Deployment Checklist

```
Before disabling local merge:
  ✔ Audit all existing local firewall rules on endpoints:
    Get-NetFirewallRule -PolicyStore ActiveStore |
        Where-Object { $_.PolicyStoreSourceType -eq 'Local' }
  ✔ Replicate any legitimate local rules into GPO
  ✔ Test with pilot group before fleet deployment
  ✔ Ensure all required apps have GPO rules created
  ⚠ Applications that self-create firewall rules at install may break
    → Test each application category before rollout
```
