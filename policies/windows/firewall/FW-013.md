# FW-013 — GPO-Pushed Firewall Rules Deployment (Full Deployment Guide)

**ID:** FW-013  
**Category:** Windows Firewall / GPO Deployment / Enterprise  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11 (Domain-joined)  
**Source:** Microsoft GPO Documentation · CIS Benchmark

---

## GPO Deployment Strategy

```
GPO Structure (recommended):

├─ GPO: Firewall-Baseline
│    ├─ Enable all profiles (FW-001)
│    ├─ Enable logging (FW-002)
│    ├─ Stealth mode (FW-005)
│    └─ Disable local rule merge (FW-011)
│
├─ GPO: Firewall-Workstations
│    ├─ Block SMB inbound (FW-003)
│    ├─ Block RDP on Public (FW-004)
│    ├─ Block WinRM inbound (FW-006)
│    ├─ Block RPC inbound (FW-007)
│    ├─ Block lateral movement ports (FW-009)
│    └─ Block mDNS/NetBIOS (FW-012)
│
└─ GPO: Firewall-Outbound
     ├─ Block malicious outbound ports (FW-008)
     └─ (Optional) IPSec rules (FW-010)
```

## PowerShell — Export and Import Firewall Policy

```powershell
# Export current firewall policy to file
netsh advfirewall export "C:\FirewallPolicy.wfw"

# Import policy on another machine
netsh advfirewall import "C:\FirewallPolicy.wfw"

# Export via PowerShell (to XML for review)
$rules = Get-NetFirewallRule | Where-Object { $_.Enabled -eq "True" }
$rules | Select DisplayName, Direction, Action, Profile, LocalPort, RemotePort |
    Export-Csv "C:\FirewallRules.csv" -NoTypeInformation

# Audit all enabled firewall rules with port details
Get-NetFirewallRule -Enabled True | ForEach-Object {
    $portFilter = $_ | Get-NetFirewallPortFilter
    $addrFilter = $_ | Get-NetFirewallAddressFilter
    [PSCustomObject]@{
        Name        = $_.DisplayName
        Direction   = $_.Direction
        Action      = $_.Action
        Profile     = $_.Profile
        Protocol    = $portFilter.Protocol
        LocalPort   = $portFilter.LocalPort
        RemotePort  = $portFilter.RemotePort
        RemoteAddr  = $addrFilter.RemoteAddress
    }
} | Export-Csv "C:\DetailedFirewallRules.csv" -NoTypeInformation
```

## Firewall Rule Conflict Resolution

```
Windows Firewall rule priority (highest to lowest):
1. Block rules always win over Allow rules
2. GPO rules always win over local rules (when merge disabled)
3. More specific rules win over less specific rules (same type)
4. Rules with authentication conditions evaluated before those without
```

## Full Deployment Checklist

```
Baseline:
  [ ] FW-001: All profiles enabled, default inbound block
  [ ] FW-002: Logging enabled, 16MB, drop+allow
  [ ] FW-005: Stealth mode (no unicast response)
  [ ] FW-011: Local rule merge disabled (Domain profile)

Lateral Movement Prevention:
  [ ] FW-003: Block SMB inbound (workstations)
  [ ] FW-004: Block RDP on Public, restrict Domain to jump server
  [ ] FW-006: Block WinRM on workstations
  [ ] FW-007: Block RPC inbound (workstations)
  [ ] FW-009: Block workstation-to-workstation lateral movement ports
  [ ] FW-012: Block mDNS + NetBIOS inbound

Outbound Control:
  [ ] FW-008: Block known malicious/legacy outbound ports

Advanced:
  [ ] FW-010: IPSec domain isolation (high-security environments)
```
