# FW-009 — Block DNS over Non-Standard Ports & DoH Enforcement

**ID:** FW-009  
**Category:** Windows Firewall / DNS Security  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft Security Baseline · CISA

---

## Description

Malware and C2 frameworks commonly abuse DNS by tunneling data over DNS queries (DNS tunneling) or using non-standard ports for DNS to bypass firewall controls. This policy restricts outbound DNS to trusted DNS servers only and blocks non-standard DNS transports.

## Registry

```
; Configure DNS-over-HTTPS (DoH) enforcement (Windows 11)
HKLM\SYSTEM\CurrentControlSet\Services\Dnscache\Parameters
  EnableAutoDoh = 2   ; 0=off, 2=on (use DoH if server supports it), 3=require DoH

; Set trusted DNS servers (via network adapter or DHCP)
; Block outbound DNS to non-authorized servers (firewall rules below)
```

## PowerShell

```powershell
# Block outbound DNS (UDP/TCP 53) to all except corporate DNS servers
$corporateDNS = "10.0.0.1","10.0.0.2"  # Internal DNS servers

# Allow DNS to corporate servers only
New-NetFirewallRule -DisplayName "Allow DNS to Corp Servers" `
    -Direction Outbound -Protocol UDP -RemotePort 53 `
    -RemoteAddress $corporateDNS `
    -Action Allow -Profile Domain -Enabled True

New-NetFirewallRule -DisplayName "Allow DNS TCP to Corp Servers" `
    -Direction Outbound -Protocol TCP -RemotePort 53 `
    -RemoteAddress $corporateDNS `
    -Action Allow -Profile Domain -Enabled True

# Block DNS to all other destinations (catches DNS tunneling to external)
New-NetFirewallRule -DisplayName "Block Unauthorized DNS" `
    -Direction Outbound -Protocol UDP -RemotePort 53 `
    -Action Block -Profile Domain -Enabled True

# Block common DNS tunneling tools (iodine uses UDP 53 with raw IP payload)
# Detection: unusually large DNS packets or high-frequency queries
# Monitor via DNS debug logging or SIEM DNS query analysis

# Configure DoH in Windows 11
Set-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Services\Dnscache\Parameters" `
    -Name EnableAutoDoh -Value 2
```

## DNS Tunneling Attack Context

```
DNS tunneling (iodine, dnscat2):
  Attacker controls DNS server for attacker-domain.com
  Malware on machine sends DNS queries:
    data.chunk1.attacker-domain.com
    data.chunk2.attacker-domain.com
  Each query carries ~100-200 bytes of payload
  Attacker's DNS server responds with encoded data
  → Full C2 channel over DNS (bypasses HTTP/HTTPS filtering)

FW-009 mitigation:
  Outbound UDP 53 only to corporate DNS
  Corporate DNS: does not resolve attacker-domain.com
  DNS queries to attacker domain → NXDOMAIN
  Tunneling blocked at DNS resolution
```
