# FW-009 — Block Lateral Movement Ports (Workstation to Workstation)

**ID:** FW-009  
**Category:** Windows Firewall / Lateral Movement Prevention  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft CISA Zero Trust · CIS Benchmark

---

## PowerShell

```powershell
# The core lateral movement prevention strategy:
# Workstations should NEVER accept inbound connections from OTHER workstations
# Only from: domain controllers, servers, management systems

# Define management subnet (adjust to your environment)
$mgmtSubnet = "10.0.1.0/24"   # IT/management subnet
$dcIP = "10.0.0.10"           # Domain Controller IP

# Block inbound SMB from non-management workstations
New-NetFirewallRule -DisplayName "Block SMB W2W" `
                   -Direction Inbound -Protocol TCP -LocalPort 445 `
                   -RemoteAddress LocalSubnet -Action Block `
                   -Profile Domain -Enabled True

# Allow SMB only from management subnet and DCs
New-NetFirewallRule -DisplayName "Allow SMB from Mgmt" `
                   -Direction Inbound -Protocol TCP -LocalPort 445 `
                   -RemoteAddress "$mgmtSubnet,$dcIP" `
                   -Action Allow -Profile Domain -Enabled True

# Block RDP workstation-to-workstation
New-NetFirewallRule -DisplayName "Block RDP W2W" `
                   -Direction Inbound -Protocol TCP -LocalPort 3389 `
                   -RemoteAddress LocalSubnet -Action Block `
                   -Profile Domain -Enabled True

# Allow RDP only from management subnet
New-NetFirewallRule -DisplayName "Allow RDP from Mgmt" `
                   -Direction Inbound -Protocol TCP -LocalPort 3389 `
                   -RemoteAddress $mgmtSubnet `
                   -Action Allow -Profile Domain -Enabled True
```

## Lateral Movement Port Reference

| Port | Protocol | Service | Block W2W |
|---|---|---|---|
| 445 | TCP | SMB | ✅ |
| 3389 | TCP | RDP | ✅ |
| 135 | TCP | RPC/DCOM | ✅ |
| 5985/5986 | TCP | WinRM | ✅ |
| 139 | TCP | NetBIOS Session | ✅ |
| 137/138 | UDP | NetBIOS Name/Datagram | ✅ |

## Description

The single most effective network-layer control for limiting ransomware blast radius is preventing workstation-to-workstation communication on lateral movement ports. Ransomware spreads via SMB, RDP, and WMI from one compromised workstation to all reachable peers. If workstations can't accept inbound SMB/RDP from each other, a single infected endpoint cannot self-replicate across the network — a $10,000 incident stays a $10,000 incident rather than becoming a $10,000,000 one.

## Impact

- ✅ Dramatically limits ransomware propagation blast radius
- ✅ Forces lateral movement through managed chokepoints (jump servers)
- ✅ Does not affect workstation access to servers (servers are not in LocalSubnet rule)
- ⚠️ Some peer-to-peer enterprise applications (VoIP, video conf) may be affected
- ℹ️ Complement with VLAN microsegmentation at the network layer for defense-in-depth
