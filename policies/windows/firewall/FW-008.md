# FW-008 — Outbound Firewall Rules Baseline (Deny Unknown Applications)

**ID:** FW-008  
**Category:** Windows Firewall / Outbound Control  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline

---

## Description

The default Windows Firewall allows all outbound traffic. For high-security environments (Public profile or sensitive workstations), an explicit outbound allowlist limits which applications can reach the network — blocking malware C2, lateral movement, and data exfiltration by unknown processes.

## Outbound Allowlist Baseline (PowerShell)

```powershell
# Step 1: Set Public profile to block all outbound (FW-001)
Set-NetFirewallProfile -Profile Public -DefaultOutboundAction Block

# Step 2: Allow specific outbound applications
# DNS
New-NetFirewallRule -DisplayName "Allow DNS Outbound" `
    -Direction Outbound -Protocol UDP -RemotePort 53 -Action Allow -Profile Any

# HTTPS (browsers, updates)
New-NetFirewallRule -DisplayName "Allow HTTPS Outbound" `
    -Direction Outbound -Protocol TCP -RemotePort 443 -Action Allow -Profile Any

# HTTP (required for some services — prefer HTTPS)
New-NetFirewallRule -DisplayName "Allow HTTP Outbound" `
    -Direction Outbound -Protocol TCP -RemotePort 80 -Action Allow -Profile Any

# Kerberos (domain auth)
New-NetFirewallRule -DisplayName "Allow Kerberos Outbound" `
    -Direction Outbound -Protocol TCP -RemotePort 88 -Action Allow -Profile Domain

# LDAP (domain queries)
New-NetFirewallRule -DisplayName "Allow LDAP Outbound" `
    -Direction Outbound -Protocol TCP -RemotePort 389 -Action Allow -Profile Domain

# SMB to file servers only (domain profile)
New-NetFirewallRule -DisplayName "Allow SMB to File Servers" `
    -Direction Outbound -Protocol TCP -RemotePort 445 `
    -RemoteAddress "10.0.0.10","10.0.0.11" `
    -Action Allow -Profile Domain

# Windows Update
New-NetFirewallRule -DisplayName "Allow Windows Update Outbound" `
    -Direction Outbound -Protocol TCP -RemotePort 443 `
    -RemoteAddress "windowsupdate.microsoft.com" `
    -Action Allow -Profile Any

# NTP (time sync)
New-NetFirewallRule -DisplayName "Allow NTP Outbound" `
    -Direction Outbound -Protocol UDP -RemotePort 123 -Action Allow -Profile Any

# ICMP (ping — optional, controlled)
New-NetFirewallRule -DisplayName "Allow ICMP Echo Outbound" `
    -Direction Outbound -Protocol ICMPv4 -IcmpType 8 `
    -Action Allow -Profile Domain,Private
```

## C2 Blocking Value

```
Common malware C2 ports (outbound block impact):
  Non-standard ports (e.g. 4444, 8888, 1337): blocked by allowlist
  Standard ports (443, 80): harder to block (C2 over HTTPS common)
  → Pair with DNS filtering (FW-009) for C2 domain blocking
  → Pair with Defender Network Protection (WDA-006) for URL/IP blocking
  → Full outbound control requires HTTPS inspection proxy
```
