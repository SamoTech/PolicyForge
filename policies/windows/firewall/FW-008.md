# FW-008 — Block Outbound to Known Malicious Ports (C2 Traffic)

**ID:** FW-008  
**Category:** Windows Firewall / Outbound Filtering / C2 Detection  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** NSA/CISA Best Practices · Microsoft Security Baseline

---

## PowerShell

```powershell
# Block outbound on commonly abused non-standard ports
# IRC (C2 channel)
New-NetFirewallRule -DisplayName "Block Outbound IRC" `
                   -Direction Outbound -Protocol TCP -RemotePort 6660-6669,7000 `
                   -Action Block -Profile Any -Enabled True

# Block Telnet outbound
New-NetFirewallRule -DisplayName "Block Outbound Telnet" `
                   -Direction Outbound -Protocol TCP -RemotePort 23 `
                   -Action Block -Profile Any -Enabled True

# Block outbound TFTP (used in some malware download chains)
New-NetFirewallRule -DisplayName "Block Outbound TFTP" `
                   -Direction Outbound -Protocol UDP -RemotePort 69 `
                   -Action Block -Profile Any -Enabled True

# Block direct outbound SMTP from workstations (spam/malware C2)
New-NetFirewallRule -DisplayName "Block Outbound SMTP from Workstations" `
                   -Direction Outbound -Protocol TCP -RemotePort 25 `
                   -Action Block -Profile Domain,Private -Enabled True

# Verify outbound block rules
Get-NetFirewallRule -Direction Outbound | Where-Object { $_.Action -eq "Block" } |
    Select DisplayName, Enabled, LocalPort, RemotePort
```

## C2 Port Quick Reference

| Port(s) | Protocol | Abuse Pattern | Block |
|---|---|---|---|
| 6660-6669, 7000 | TCP | IRC-based C2 (classic botnets) | ✅ All profiles |
| 23 | TCP | Telnet (cleartext creds, IoT bots) | ✅ All profiles |
| 69 | UDP | TFTP malware download | ✅ All profiles |
| 25 | TCP | Direct SMTP (spam/malware relay) | ✅ Domain/Private (workstations) |
| 4444, 1337, 31337 | TCP | Common Metasploit/reverse shell ports | ✅ All profiles |
| 9001, 9030 | TCP | Tor relay ports | ✅ Corporate environments |

## Description

Outbound firewall filtering on workstations blocks malware command-and-control (C2) communication on commonly abused ports. Most enterprise firewall policies default-allow all outbound traffic, meaning malware beacons can phone home freely. While full outbound blocking is operationally complex, blocking a small set of known-malicious or legacy ports (IRC, Telnet, TFTP, direct SMTP) provides meaningful C2 disruption with minimal legitimate application impact.

## Impact

- ✅ Disrupts IRC-based botnet C2 channels
- ✅ Blocks direct spam relay from infected workstations
- ✅ Detects anomalous outbound traffic via firewall logs
- ⚠️ Will not block C2 over HTTPS (443) — requires proxy/EDR for that
- ℹ️ For full outbound control, move to explicit-deny outbound with application-specific allow rules
