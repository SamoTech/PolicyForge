# FW-005 — Enable Firewall Stealth Mode (Prevent Host Discovery)

**ID:** FW-005  
**Category:** Windows Firewall / Stealth Mode / Host Discovery  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ [Profile] Properties
                      └─ Settings > Customize
                           ├─ Display a notification: No
                           └─ Allow unicast response: No
```

## Registry

```
; Public Profile (most important for stealth)
HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\PublicProfile
  DisableUnicastResponsesToMulticastBroadcast = 1
  DisableNotifications = 1

HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall\DomainProfile
  DisableUnicastResponsesToMulticastBroadcast = 1
```

## PowerShell

```powershell
# Disable unicast responses to multicast/broadcast (prevents host discovery)
Set-NetFirewallProfile -Profile Domain,Private,Public `
                       -NotifyOnListen False

Set-NetFirewallProfile -Profile Public `
                       -AllowUnicastResponseToMulticast False

# Verify
Get-NetFirewallProfile | Select Name, NotifyOnListen, AllowUnicastResponseToMulticast
```

## Description

Stealth mode prevents the machine from responding to unsolicited broadcast and multicast packets. Attackers use network discovery tools (Nmap, Masscan, NetBIOS scans) to enumerate hosts on a network. When stealth mode is enabled, the host simply doesn't respond to these probes, making it invisible to casual network scanners. This is especially important on the Public profile (untrusted networks like hotel Wi-Fi).

## Impact

- ✅ Host doesn't respond to broadcast/multicast network discovery probes
- ✅ Reduces attack surface visibility to network scanners
- ✅ Suppresses firewall notification popups (less user confusion)
- ⚠️ May interfere with some network troubleshooting tools
- ⚠️ DHCP renewal uses broadcast — this is handled separately and not affected
