# FW-005 — Block Inbound RDP from Internet (Public Profile)

**ID:** FW-005  
**Category:** Windows Firewall / Remote Access Restriction  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ Inbound Rules
                      ├─ Allow RDP (TCP 3389) — Domain profile only
                      │    Scope: Local subnet or specific admin IPs
                      ├─ Block RDP (TCP 3389) — Private profile
                      │    (or restrict to specific IPs)
                      └─ Block RDP (TCP 3389) — Public profile (explicit block)
```

## PowerShell

```powershell
# Remove default RDP rule for all profiles
Get-NetFirewallRule -DisplayName "*Remote Desktop*" | Disable-NetFirewallRule

# Create restricted RDP rule — Domain profile, corp subnet only
New-NetFirewallRule -DisplayName "PolicyForge: RDP Domain Admins Only" `
    -Direction Inbound -Protocol TCP -LocalPort 3389 `
    -RemoteAddress "10.0.0.0/8" `
    -Action Allow -Profile Domain -Enabled True

# Explicit block on Public profile
New-NetFirewallRule -DisplayName "PolicyForge: Block RDP Public" `
    -Direction Inbound -Protocol TCP -LocalPort 3389 `
    -Action Block -Profile Public -Enabled True

# Change RDP port to non-standard (additional obscurity)
# HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp
# PortNumber = 3389 (change to non-standard, not a security control by itself)
```

## RDP Attack Statistics Context

```
RDP (port 3389) is the #1 most scanned port on the internet.
Shodan search: port:3389 → millions of exposed RDP endpoints

Common RDP attacks:
  BlueKeep (CVE-2019-0708): unauthenticated RCE, pre-auth
  DejaBlue (CVE-2019-1181/1182): unauthenticated RCE
  NLA bypass: credential relay attacks
  Brute force: automated password spraying (ACC-006 lockout helps)
  MitM: RDP downgrade attacks without NLA

With FW-005:
  Port 3389 closed on Public/Internet profile
  → Internet scanners cannot reach RDP
  → Even unpatched RDP is not exploitable remotely
  → VPN required before RDP access (preferred)
```

## Best Practice: RDP via VPN Only

```
Recommended architecture:
  Internet → VPN Gateway → Internal Network → RDP to machine

  Benefits:
  ✔ RDP never exposed to internet
  ✔ VPN provides pre-authentication
  ✔ MFA enforced at VPN layer
  ✔ Encrypted tunnel before RDP session
```
