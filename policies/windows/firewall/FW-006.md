# FW-006 — Block WinRM (Port 5985/5986) Inbound on Non-Server Endpoints

**ID:** FW-006  
**Category:** Windows Firewall / Remote Management  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark · Microsoft Security Baseline

---

## PowerShell

```powershell
# Block WinRM HTTP (5985) and HTTPS (5986) inbound on workstations
New-NetFirewallRule -DisplayName "Block WinRM HTTP Inbound" `
                   -Direction Inbound -Protocol TCP -LocalPort 5985 `
                   -Action Block -Profile Any -Enabled True `
                   -Description "Block PowerShell Remoting on workstations"

New-NetFirewallRule -DisplayName "Block WinRM HTTPS Inbound" `
                   -Direction Inbound -Protocol TCP -LocalPort 5986 `
                   -Action Block -Profile Any -Enabled True `
                   -Description "Block WinRM HTTPS on workstations"

# If WinRM is NOT needed at all, disable the service:
Set-Service WinRM -StartupType Disabled
Stop-Service WinRM

# If WinRM IS needed (management), restrict to IT jump server:
New-NetFirewallRule -DisplayName "Allow WinRM from Jump Server" `
                   -Direction Inbound -Protocol TCP -LocalPort 5985 `
                   -RemoteAddress "10.0.1.10" `
                   -Action Allow -Profile Domain -Enabled True

# Verify
Get-NetFirewallRule | Where-Object { $_.LocalPort -in @(5985, 5986) } |
    Select DisplayName, Direction, Action, Enabled
```

## Description

WinRM (Windows Remote Management) enables PowerShell Remoting — a powerful lateral movement tool used by attackers to run commands on remote endpoints after gaining initial access. Ransomware operators and APT groups heavily rely on WinRM for remote command execution once inside a network. Workstations have no legitimate need to accept WinRM connections — block it entirely and restrict servers to management subnet IPs only.

## Ports Covered

| Port | Protocol | Service |
|---|---|---|
| 5985 | TCP | WinRM HTTP (PowerShell Remoting) |
| 5986 | TCP | WinRM HTTPS (PowerShell Remoting Secure) |
