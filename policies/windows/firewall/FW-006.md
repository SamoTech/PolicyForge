# FW-006 — Block Inbound SMB from Internet & Restrict Lateral Movement

**ID:** FW-006  
**Category:** Windows Firewall / SMB Restriction  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ Inbound Rules
                      ├─ Block: TCP 445 (SMB) — Public profile
                      ├─ Block: TCP 139 (SMB over NetBIOS) — Public profile
                      └─ Restrict: TCP 445 — Domain profile to specific servers only
```

## PowerShell

```powershell
# Block SMB on Public profile (internet-facing)
New-NetFirewallRule -DisplayName "PolicyForge: Block SMB Public Inbound" `
    -Direction Inbound -Protocol TCP -LocalPort 445 `
    -Action Block -Profile Public -Enabled True

# Restrict SMB on Domain profile to specific file servers only
# (Reduces lateral movement surface)
New-NetFirewallRule -DisplayName "PolicyForge: Restrict SMB Domain Inbound" `
    -Direction Inbound -Protocol TCP -LocalPort 445 `
    -RemoteAddress "10.0.0.10","10.0.0.11"  `  # File server IPs only
    -Action Allow -Profile Domain -Enabled True

# Block all other SMB inbound on Domain
New-NetFirewallRule -DisplayName "PolicyForge: Block SMB Domain Default" `
    -Direction Inbound -Protocol TCP -LocalPort 445 `
    -Action Block -Profile Domain -Enabled True

# Check existing SMB-related firewall rules
Get-NetFirewallRule | Where-Object { $_.DisplayName -match 'SMB|File' } |
    Select DisplayName, Direction, Action, Enabled, Profile
```

## EternalBlue Context

```
EternalBlue (CVE-2017-0144) — SMB exploit used in WannaCry / NotPetya:
  Requires: TCP 445 reachable
  WannaCry spread: scanned for port 445 on internet + internal network
  NotPetya spread: internal propagation via 445 after initial compromise

With port 445 blocked on Public profile:
  → EternalBlue-class attacks cannot reach machine from internet
  → Even unpatched systems not exploitable externally

Lateral movement restriction (Domain profile):
  Workstations don't need to accept SMB from other workstations
  Allow 445 only FROM specific file servers
  → Lateral movement via SMB between workstations blocked
  → PsExec, WMI-SMB lateral movement significantly impeded
```
