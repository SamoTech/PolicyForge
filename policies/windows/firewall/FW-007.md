# FW-007 — Restrict Inbound DCOM / RPC (Port 135) on Workstations

**ID:** FW-007  
**Category:** Windows Firewall / RPC / DCOM  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark · Microsoft Security Baseline

---

## PowerShell

```powershell
# Block RPC Endpoint Mapper inbound on workstations (Public + Private)
New-NetFirewallRule -DisplayName "Block RPC Inbound Workstations" `
                   -Direction Inbound -Protocol TCP -LocalPort 135 `
                   -Action Block -Profile Public,Private -Enabled True `
                   -Description "Block DCOM/RPC inbound on workstations"

# Block dynamic RPC port range on Public
New-NetFirewallRule -DisplayName "Block Dynamic RPC Inbound Public" `
                   -Direction Inbound -Protocol TCP -LocalPort "49152-65535" `
                   -Action Block -Profile Public -Enabled True

# Verify
Get-NetFirewallRule | Where-Object { $_.LocalPort -like "*135*" } |
    Select DisplayName, Action, Profile, Enabled
```

## Description

DCOM/RPC (TCP 135 + dynamic ports 49152-65535) is exploited by several critical Windows vulnerabilities including MS03-026, MS03-039 (historical) and more recently CVE-2022-26809 (RCE via RPC, CVSS 9.8). Workstations have no legitimate inbound RPC requirement from peers. Blocking TCP 135 inbound on Public and Private profiles removes a significant remote code execution attack surface while preserving domain controller communication on the Domain profile.

## Impact

- ✅ Blocks exploitation of RPC-based remote code execution vulnerabilities
- ✅ Prevents DCOM-based lateral movement techniques
- ⚠️ Some management tools (WMI remote queries) require RPC — restrict by source IP on Domain profile
- ℹ️ Domain profile: allow only from trusted management systems
