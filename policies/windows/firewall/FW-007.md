# FW-007 — Connection Security Rules (IPsec for Domain Isolation)

**ID:** FW-007  
**Category:** Windows Firewall / IPsec Connection Security  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** Microsoft Security Baseline · NIST 800-77

---

## Description

Connection Security Rules (IPsec) in Windows Firewall enforce authenticated and/or encrypted communication between machines. Domain Isolation uses IPsec to ensure only domain-authenticated machines can communicate with each other — rogue or guest machines cannot reach domain resources even if physically on the network.

## Policy Path

```
Computer Configuration
  └─ Windows Settings
       └─ Security Settings
            └─ Windows Defender Firewall with Advanced Security
                 └─ Connection Security Rules
                      └─ New Rule → Isolation
                           ├─ Requirements: Request authentication for inbound and outbound
                           ├─ Authentication method: Computer and User (Kerberos V5)
                           └─ Profiles: Domain
```

## PowerShell

```powershell
# Create domain isolation connection security rule (request mode)
New-NetIPsecRule -DisplayName "PolicyForge: Domain Isolation" `
    -InboundSecurity Request `
    -OutboundSecurity Request `
    -Phase1AuthSet (Get-NetIPsecPhase1AuthSet -DisplayName "Domain Kerberos Auth" -ErrorAction SilentlyContinue)?.InstanceID `
    -Profile Domain `
    -Enabled True

# Create Kerberos authentication proposal
$authProposal = New-NetIPsecAuthProposal -Machine -Kerberos
$authSet = New-NetIPsecPhase1AuthSet -DisplayName "Domain Kerberos Auth" `
    -Proposal $authProposal

# Apply isolation rule with Kerberos
New-NetIPsecRule -DisplayName "PolicyForge: Domain Isolation" `
    -InboundSecurity Request -OutboundSecurity Request `
    -Phase1AuthSet $authSet.InstanceID `
    -Profile Domain

# View active connection security rules
Get-NetIPsecRule | Select DisplayName, InboundSecurity, OutboundSecurity, Enabled

# View active IPsec main mode SAs
Get-NetIPsecMainModeSA | Select LocalAddress, RemoteAddress, AuthenticationMethod
```

## Domain Isolation Architecture

```
Without Domain Isolation:
  Any machine on network (domain or rogue) can:
  → Connect to SMB shares
  → Authenticate to domain services
  → Send traffic to any machine

With Domain Isolation (IPsec Request mode):
  Domain machine → Domain machine:
  → IPsec Kerberos authentication negotiated
  → Encrypted and authenticated tunnel
  → Communication succeeds

  Rogue machine → Domain machine:
  → Requests IPsec negotiation
  → Cannot provide valid Kerberos ticket
  → Falls through to unauthenticated (if Request, not Require)
  → With Require mode: communication completely blocked

Modes:
  Request: Try IPsec, fall back to clear if peer doesn't support
  Require: IPsec mandatory, connection fails without it
```
