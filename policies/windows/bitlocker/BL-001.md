# BL-001 — Require TPM + PIN Pre-Boot Authentication

**Category:** BitLocker / Pre-Boot Authentication
**Risk Level:** Critical
**OS:** Windows 10+, Windows 11, Server 2016+
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

> ⚠️ **WARNING**: BitLocker encryption is a destructive, irreversible operation. If the recovery key is lost, **all data on the drive will be permanently unrecoverable**. Always back up recovery keys to Active Directory or Azure AD before enabling BitLocker. Requires TPM 1.2+ (preferably TPM 2.0).

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      ├─ Require additional authentication at startup: Enabled
                      │    ├─ Allow BitLocker without a compatible TPM: Not configured
                      │    ├─ TPM startup: Do not allow TPM
                      │    ├─ TPM startup PIN: Require startup PIN with TPM
                      │    ├─ TPM startup key: Do not allow startup key with TPM
                      │    └─ TPM startup key and PIN: Do not allow startup key and PIN with TPM
                      └─ Configure minimum PIN length for startup: Enabled → 6 digits
```

## PowerShell

```powershell
# Check BitLocker status
Get-BitLockerVolume -MountPoint C: | Select MountPoint, VolumeStatus,
    EncryptionPercentage, KeyProtector, EncryptionMethod

# Check TPM status
Get-Tpm | Select TpmPresent, TpmReady, TpmEnabled, TpmActivated,
    TpmOwned, ManufacturerVersion

# Verify registry settings
$fvePath = 'HKLM:\SOFTWARE\Policies\Microsoft\FVE'
@('EnableBDEWithNoTPM','UseTPM','UseTPMPIN','UseTPMKey','UseTPMKeyPIN','MinimumPIN','UseEnhancedPin') | ForEach-Object {
    $v = Get-ItemPropertyValue -Path $fvePath -Name $_ -ErrorAction SilentlyContinue
    [PSCustomObject]@{ ValueName=$_; Value=$v }
}
```

| `./Device/Vendor/MSFT/BitLocker/SystemDrivesMinimumPINLength` | `6` | Integer |

## Impact

Requiring a PIN at boot prevents a stolen or seized device from being decrypted even if the TPM is present and intact. Without a PIN, BitLocker with TPM-only protection can be bypassed by moving the drive to a system with the same TPM PCR values in certain attack scenarios.

## Use Cases

- Laptop/endpoint fleet hardening for mobile workers
- High-assurance environments (government, finance, healthcare)
- Meeting CIS Level 2 and DISA STIG requirements
