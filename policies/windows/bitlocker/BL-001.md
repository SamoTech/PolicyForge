# BL-001 — Require BitLocker on OS Drive (XTS-AES 256)

**ID:** BL-001  
**Category:** BitLocker / Encryption / OS Drive  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1511+, Windows 11  
**Source:** Microsoft BitLocker ADMX · CIS Benchmark Windows 11 v3.0

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      ├─ Require additional authentication at startup → Enabled
                      └─ Choose drive encryption method and cipher strength
                           └─ OS drives: XTS-AES 256-bit
```

## Registry

```
Key:   HKLM\SOFTWARE\Policies\Microsoft\FVE
Value: EncryptionMethodWithXtsFdv
Type:  REG_DWORD
Data:  7  (XTS-AES 256)

; Encryption method values:
; 3 = AES-CBC 128  (legacy, avoid)
; 4 = AES-CBC 256  (legacy, avoid)
; 6 = XTS-AES 128  (acceptable)
; 7 = XTS-AES 256  (recommended)
```

## PowerShell

```powershell
# Enable BitLocker on C: with TPM + PIN
Enable-BitLocker -MountPoint "C:" `
                 -EncryptionMethod XtsAes256 `
                 -TpmAndPinProtector `
                 -Pin (ConvertTo-SecureString "YourPIN" -AsPlainText -Force)

# Check status
Get-BitLockerVolume -MountPoint "C:" | Select MountPoint, VolumeStatus, EncryptionMethod, ProtectionStatus

# Add recovery key protector
Add-BitLockerKeyProtector -MountPoint "C:" -RecoveryPasswordProtector
```

## Intune CSP

```
OMA-URI: ./Device/Vendor/MSFT/BitLocker/EncryptionMethodByDriveType
Type:    String
Value:   <enabled/><data id="EncryptionMethodWithXtsFdv" value="7"/>
```

## Description

Forces BitLocker full-disk encryption on the OS drive using XTS-AES 256-bit cipher — the strongest encryption mode available in Windows. XTS mode provides better performance and security than CBC mode on SSDs and NVMe drives. This policy ensures all new BitLocker-enabled drives use 256-bit encryption rather than the weaker 128-bit default.

## Impact

- ✅ Strongest available encryption for OS drives
- ✅ Protects against offline attacks, stolen device data extraction
- ✅ XTS-AES is resistant to ciphertext manipulation attacks vs CBC
- ⚠️ Does NOT re-encrypt already-encrypted drives (use `manage-bde -reprotect` to change cipher on existing volumes)
- ℹ️ Encryption does not affect performance meaningfully on modern NVMe hardware

## Use Cases

- Laptop fleet protection (theft scenario)
- GDPR/HIPAA compliance (data-at-rest encryption requirement)
- Air-gapped environment endpoint hardening
- Government/classified data endpoint baseline
