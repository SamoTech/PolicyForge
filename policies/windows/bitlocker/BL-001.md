# BL-001 — Require TPM + PIN Pre-Boot Authentication

**ID:** BL-001  
**Category:** BitLocker / Pre-Boot Authentication  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      ├─ Require additional authentication at startup: Enabled
                      │    ├─ Allow BitLocker without a compatible TPM: Not configured
                      │    ├─ TPM startup: Do not allow TPM
                      │    ├─ TPM startup PIN: Require startup PIN with TPM
                      │    ├─ TPM startup key: Do not allow startup key with TPM
                      │    └─ TPM startup key and PIN: Do not allow startup key and PIN with TPM
                      └─ Configure minimum PIN length for startup: Enabled → 6 digits
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\FVE
  EnableBDEWithNoTPM          = 0   ; Require TPM
  UseTPM                      = 0   ; Do not allow TPM-only (no PIN)
  UseTPMPIN                   = 1   ; Require TPM + PIN
  UseTPMKey                   = 0   ; Do not allow TPM + key only
  UseTPMKeyPIN                = 0   ; Do not allow TPM + key + PIN
  MinimumPIN                  = 6   ; Minimum PIN length
  UseEnhancedPin              = 1   ; Allow full alphanumeric PIN
```

## PowerShell

```powershell
# Check BitLocker status
Get-BitLockerVolume -MountPoint C: | Select MountPoint, VolumeStatus,
    EncryptionPercentage, KeyProtector, EncryptionMethod

# Check TPM status
Get-Tpm | Select TpmPresent, TpmReady, TpmEnabled, TpmActivated,
    TpmOwned, ManufacturerVersion

# Enable BitLocker with TPM + PIN
$securePin = Read-Host "Enter PIN" -AsSecureString
Enable-BitLocker -MountPoint "C:" `
    -EncryptionMethod XtsAes256 `
    -TpmAndPinProtector $securePin

# Verify protector type
(Get-BitLockerVolume C:).KeyProtector | Select KeyProtectorType
```

## TPM-Only vs TPM+PIN Attack Comparison

```
TPM-Only (no PIN) — VULNERABLE:
  Scenario: Laptop stolen
  1. Boot laptop normally (TPM releases key automatically)
  2. BitLocker unlocks WITHOUT any user interaction
  3. Windows boots to login screen
  4. Attacker can attempt: brute force login, boot from USB,
     CVE-based pre-auth exploits, PCR manipulation attacks
  5. Data accessible with physical access alone

TPM + PIN — PROTECTED:
  Scenario: Same stolen laptop
  1. Boot laptop → BitLocker PIN prompt appears PRE-OS
  2. Without correct PIN → TPM refuses to release FVEK
  3. Drive remains fully encrypted
  4. Attacker has encrypted drive + no PIN = no data
  5. Even direct NAND flash reading yields only ciphertext
```

## Pre-Boot PIN Best Practices

```
✔ Minimum 6 digits (numeric)
✔ Prefer alphanumeric PIN (UseEnhancedPin = 1) for higher entropy
✔ Communicate PIN separately from device at provisioning
✔ Store PIN recovery documentation securely (NOT with device)
✔ Enforce PIN via GPO — do not allow users to remove it
❌ Never use TPM-only on laptops/mobile devices
❌ Never store PIN in recovery documentation alongside recovery key
```
