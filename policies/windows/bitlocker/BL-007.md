# BL-007 — Configure BitLocker Network Unlock (Domain Environments)

**ID:** BL-007  
**Category:** BitLocker / Network Unlock / Enterprise  
**Risk Level:** 🟠 Medium  
**OS:** Windows 8+, Windows 10, Windows 11 (domain-joined)  
**Source:** Microsoft BitLocker Network Unlock Documentation

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      └─ Allow Secure Boot for integrity validation → Enabled

(Network Unlock is configured via WDS/DHCP server, not Group Policy alone)
```

## Registry

```
Key:   HKLM\SOFTWARE\Policies\Microsoft\FVE
Value: OSManageNKP = 1 (DWORD) — Enable Network Key Protector management
```

## PowerShell

```powershell
# Add Network Unlock protector to existing BitLocker volume
Add-BitLockerKeyProtector -MountPoint "C:" -ADAccountOrGroupProtector -ADAccountOrGroup "Domain Computers"

# Check protector types
(Get-BitLockerVolume -MountPoint "C:").KeyProtector | Select KeyProtectorType, KeyProtectorId

# Network Unlock requires:
# 1. WDS server with BitLocker Network Unlock provider
# 2. DHCP server on same subnet
# 3. Network Unlock certificate deployed to WDS
# 4. Client UEFI firmware with DHCP driver
```

## Description

Network Unlock automatically unlocks BitLocker-protected OS drives when a domain-joined machine boots on the corporate network. When the machine is on the trusted corporate network (identified via DHCP), BitLocker unlocks automatically without PIN entry. When off the network (e.g., stolen laptop), BitLocker requires the PIN or recovery key. This allows unattended reboots for servers and managed workstations while maintaining security for off-network devices.

## Infrastructure Requirements

| Component | Requirement |
|---|---|
| Server | Windows Server 2012+ with WDS role |
| Certificate | Network Unlock certificate deployed to WDS |
| DHCP | DHCP server on corporate subnet (option 245) |
| Client firmware | UEFI with PXE/DHCP driver |
| Domain | Machine must be domain-joined |

## Impact

- ✅ Enables unattended reboots on corporate network (servers, managed PCs)
- ✅ Still requires PIN/recovery when off corporate network
- ✅ Zero friction for end users in office
- ⚠️ Requires WDS server infrastructure
- ⚠️ Attacker on corporate network could potentially boot the machine
- ℹ️ Combine with 802.1X NAC for stronger network trust validation
