# BL-007 — BitLocker Network Unlock (Domain-Joined Desktops)

**ID:** BL-007  
**Category:** BitLocker / Network Unlock  
**Risk Level:** 🟡 Medium  
**OS:** Windows 8+, Windows 10+, Server 2012+  
**Source:** Microsoft BitLocker Documentation

---

## Description

BitLocker Network Unlock allows domain-joined desktop machines to automatically unlock their BitLocker-protected OS drive at boot when connected to the corporate wired network — without requiring a user to enter a PIN. The unlock is performed by the WDS (Windows Deployment Services) server using a certificate-based key exchange.

## Architecture

```
Network Unlock flow:
  1. Machine boots on corporate wired network
  2. BitLocker sends encrypted unlock request via DHCP (UDP/67)
  3. WDS server receives request, decrypts using network unlock cert
  4. WDS sends back volume unlock key
  5. BitLocker unlocks OS drive automatically
  6. Machine boots without PIN

Security model:
  ✅ Machine must be on trusted wired network (DHCP)
  ✅ Machine must have valid network unlock protector
  ✅ WDS server must hold valid private key
  ❌ NOT secure for laptops (can be on any network)
  ❌ NOT a replacement for PIN on mobile devices
```

## When to Use Network Unlock

| Scenario | Network Unlock Appropriate? |
|---|---|
| Desktop PCs in locked server room | ✅ Yes |
| Kiosk machines on isolated VLAN | ✅ Yes |
| Headless servers (no keyboard) | ✅ Yes (combined with TPM) |
| Laptops / mobile devices | ❌ No — use TPM+PIN (BL-001) |
| Home workers / VPN machines | ❌ No |
| Machines that travel | ❌ No |

## PowerShell — Add Network Unlock Protector

```powershell
# Add network unlock certificate protector
# (Certificate must be configured on WDS server first)
Add-BitLockerKeyProtector -MountPoint C: -ADAccountOrGroupProtector

# View all protectors
(Get-BitLockerVolume C:).KeyProtector | Select KeyProtectorType, KeyProtectorId

# Network unlock works alongside TPM protector
# Desktop with both: auto-unlocks on corp network, PIN required off-network
```
