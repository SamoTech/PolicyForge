# BL-008 — Store BitLocker Recovery Keys in Active Directory / Entra ID

**ID:** BL-008  
**Category:** BitLocker / Recovery Key Management  
**Risk Level:** 🔴 Critical (if not configured — keys get lost)  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft BitLocker ADMX · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      └─ Choose how BitLocker-protected OS drives can be recovered
                           ├─ Enabled
                           ├─ Save BitLocker recovery info to AD DS: Enabled
                           ├─ Do not enable BitLocker until recovery info is stored in AD DS: Enabled
                           └─ Recovery key: Allow 256-bit recovery key
```

## Registry

```
Key:   HKLM\SOFTWARE\Policies\Microsoft\FVE
Value: OSRecovery                  = 1 — Enable recovery
Value: OSManageRecovery            = 1 — Manage recovery options
Value: OSRecoveryPassword          = 2 — Allow 48-digit recovery password
Value: OSRecoveryKey               = 2 — Allow 256-bit recovery key
Value: OSActiveDirectoryBackup     = 1 — Backup to AD DS
Value: OSRequireActiveDirectoryBackup = 1 — Do not encrypt until backed up to AD
Value: OSActiveDirectoryInfoToStore = 1 — Store recovery password in AD
```

## PowerShell

```powershell
# Backup existing recovery key to Azure AD / Entra ID
$BLV = Get-BitLockerVolume -MountPoint "C:"
ForEach ($Protector in $BLV.KeyProtector) {
    If ($Protector.KeyProtectorType -eq "RecoveryPassword") {
        BackupToAAD-BitLockerKeyProtector -MountPoint "C:" `
                                          -KeyProtectorId $Protector.KeyProtectorId
        Write-Host "Recovery key backed up to Azure AD: $($Protector.KeyProtectorId)"
    }
}

# Backup to on-prem Active Directory
Backup-BitLockerKeyProtector -MountPoint "C:" `
                              -KeyProtectorId $Protector.KeyProtectorId

# Verify backup
Get-BitLockerVolume "C:" | Select -ExpandProperty KeyProtector
```

## Intune CSP (Entra ID Backup)

```
OMA-URI: ./Device/Vendor/MSFT/BitLocker/SystemDrivesRecoveryOptions
Value:   <enabled/>
         <data id="OSAllowDRA_Name" value="true"/>
         <data id="OSRecoveryPasswordUsageDropDown_Name" value="2"/>
         <data id="OSHideRecoveryPage_Name" value="false"/>
         <data id="OSActiveDirectoryBackup_Name" value="true"/>
         <data id="OSRequireActiveDirectoryBackup_Name" value="true"/>
```

## Description

Configures BitLocker to automatically backup recovery keys to Active Directory (on-prem) or Entra ID (cloud). Without this policy, recovery keys may exist only on the user's device or printout — easily lost. This policy blocks BitLocker from activating until the recovery key is successfully escrowed, ensuring IT always has a recovery path.

## Impact

- ✅ Centralized recovery key management for IT
- ✅ Prevents permanent data loss from forgotten PINs
- ✅ Blocks BitLocker activation until key is safely backed up
- ⚠️ AD DS must have BitLocker schema extensions (included in Windows Server 2012+)
- ⚠️ Recovery keys in AD should be access-controlled (not all IT staff need access)
- ℹ️ For Entra ID (Intune): keys visible in Entra ID portal → Devices → [device] → Recovery keys
