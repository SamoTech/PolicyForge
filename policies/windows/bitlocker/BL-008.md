# BL-008 — Configure BitLocker on Servers (OS + Data Volumes)

**ID:** BL-008  
**Category:** BitLocker / Server Encryption  
**Risk Level:** 🟠 High  
**OS:** Windows Server 2016+  
**Source:** Microsoft Security Baseline · CIS Benchmark Server

---

## Description

BitLocker on servers is primarily used to protect against **physical access threats** — a decommissioned server, stolen drive, or data center breach. For servers that don’t have a keyboard (headless), Network Unlock (BL-007) is the standard unlock mechanism.

## PowerShell — Server BitLocker Deployment

```powershell
# Install BitLocker feature on Server
Install-WindowsFeature BitLocker -IncludeAllSubFeature -IncludeManagementTools -Restart

# Check TPM on server
Get-Tpm

# Enable BitLocker on OS drive (TPM only for headless server)
Enable-BitLocker -MountPoint C: `
    -EncryptionMethod XtsAes256 `
    -TpmProtector

# Add recovery password and backup to AD
Add-BitLockerKeyProtector -MountPoint C: -RecoveryPasswordProtector
$id = (Get-BitLockerVolume C:).KeyProtector |
    Where-Object {$_.KeyProtectorType -eq 'RecoveryPassword'} |
    Select-Object -ExpandProperty KeyProtectorId
Backup-BitLockerKeyProtector -MountPoint C: -KeyProtectorId $id

# Encrypt data volumes (auto-unlock tied to OS drive)
Enable-BitLocker -MountPoint D: -EncryptionMethod XtsAes256 -RecoveryPasswordProtector
Enable-BitLockerAutoUnlock -MountPoint D:

# Verify
Get-BitLockerVolume | Select MountPoint, VolumeStatus, KeyProtector
```

## Server Considerations

```
✅ Use TPM-only for headless servers + Network Unlock for remote boot
✅ Auto-unlock data volumes (D:, E:) tied to encrypted C:
✅ Always backup recovery keys to AD before enabling
✅ Test recovery process in staging before production
⚠️ BitLocker on storage servers with many drives: test I/O impact
⚠️ Hyper-V hosts: encrypt host OS, VHDs protected by guest OS BitLocker
⚠️ SQL Server: test performance on data volumes before production
```
