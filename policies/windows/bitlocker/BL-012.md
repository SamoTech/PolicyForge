# BL-012 — Disable BitLocker on Hardware Encrypted Drives (eDrive)

**ID:** BL-012  
**Category:** BitLocker / Hardware Encryption / eDrive  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1709+, Windows 11  
**Source:** Microsoft Security Advisory ADV180028 · CVE-2018-12037 / CVE-2018-12038

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      └─ Configure use of hardware-based encryption for operating system drives
                           ├─ Enabled
                           └─ Use BitLocker software-based encryption when hardware encryption is not available: Enabled
                              (Effectively forces software encryption)
```

## Registry

```
Key:   HKLM\SOFTWARE\Policies\Microsoft\FVE
Value: OSHardwareEncryption
Type:  REG_DWORD
Data:  0  (0 = Disable hardware encryption, force software)
```

## PowerShell

```powershell
# Force software encryption (disable eDrive hardware encryption)
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\FVE" `
                 -Name "OSHardwareEncryption" -Value 0 -Type DWord
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\FVE" `
                 -Name "FDVHardwareEncryption" -Value 0 -Type DWord
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\FVE" `
                 -Name "RDVHardwareEncryption" -Value 0 -Type DWord

# Check current encryption type:
manage-bde -status C: | findstr "Encryption Method"
# 'XTS-AES 256' = software encryption (good)
# 'Hardware Encryption' = eDrive (vulnerable)

# Re-encrypt if currently using hardware encryption:
manage-bde -off C:
# Wait for decryption, then re-enable with software encryption:
Enable-BitLocker -MountPoint "C:" -EncryptionMethod XtsAes256 -TpmAndPinProtector -Pin (Read-Host -AsSecureString "PIN")
```

## Description

Forces BitLocker to use Windows software-based encryption instead of hardware-based encryption (eDrive/TCG OPAL). In 2018, researchers (Radboud University) discovered critical vulnerabilities in hardware encryption implementations of major SSD vendors (Samsung, Crucial/Micron, Western Digital) — CVE-2018-12037 and CVE-2018-12038 — where the encryption password was either ignored, stored in plaintext, or bypassable. Microsoft issued ADV180028 recommending disabling hardware encryption. Software BitLocker (XTS-AES 256) is cryptographically verified and not affected.

## CVE References

|---|---|---|

| CVE-2018-12038 | Samsung 840 EVO | Empty password unlocks |
| ADV180028 | Crucial MX100/MX200/MX300 | Master password bypass |

## Impact

- ✅ Eliminates eDrive hardware encryption vulnerabilities
- ✅ Uses cryptographically verified AES-XTS 256 software encryption
- ⚠️ Slight performance impact vs hardware encryption on very old hardware (negligible on modern NVMe)
- ⚠️ Requires full re-encryption if drive currently uses hardware encryption
