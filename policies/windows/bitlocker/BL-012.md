# BL-012 — Disable BitLocker Suspension & Suspend Controls

**ID:** BL-012  
**Category:** BitLocker / Policy Enforcement  
**Risk Level:** 🟠 High  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** Microsoft Security Baseline

---

## Description

BitLocker can be temporarily suspended (protectors turned off, drive still encrypted but key stored in cleartext on disk). This is used legitimately during firmware/BIOS updates. Attackers can abuse suspension to extract the unprotected FVEK from the unencrypted metadata area of the disk.

## PowerShell — Monitor BitLocker Suspension

```powershell
# Check for suspended BitLocker volumes
Get-BitLockerVolume | Where-Object { $_.ProtectionStatus -eq 'Off' -and $_.VolumeStatus -eq 'FullyEncrypted' } |
    Select MountPoint, ProtectionStatus, VolumeStatus, KeyProtector

# If suspended: re-enable protection immediately
Resume-BitLocker -MountPoint C:

# Alert script: check all machines for suspended BitLocker
$computers = Get-ADComputer -Filter * | Select -ExpandProperty Name
foreach ($computer in $computers) {
    $vol = Invoke-Command -ComputerName $computer -ScriptBlock {
        Get-BitLockerVolume C: | Select MountPoint, ProtectionStatus
    } -ErrorAction SilentlyContinue
    if ($vol.ProtectionStatus -eq 'Off') {
        Write-Warning "ALERT: BitLocker SUSPENDED on $computer"
    }
}
```

## Suspension Attack

```
BitLocker suspension attack:
  1. Attacker or malware runs: Suspend-BitLocker -MountPoint C: -RebootCount 0
  2. BitLocker protection disabled — drive still shows as encrypted
  3. FVEK now stored in cleartext in Volume Master Key (VMK) area
  4. Read VMK from disk metadata: manage-bde -protectors -get C:
  5. Extract FVEK → full drive decryption without PIN/TPM

Detection:
  ProtectionStatus = Off + VolumeStatus = FullyEncrypted
  → BitLocker is SUSPENDED (not off, but unprotected)
  → Alert immediately and resume protection

Authorized suspension (firmware update):
  → Suspend via MECM/Intune task sequence only
  → Auto-resumes after reboot count expires
  → Monitor via SIEM for unauthorized suspension
```
