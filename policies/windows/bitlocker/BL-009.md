# BL-009 — Harden BitLocker Against Pre-Boot Attacks (DMA Protection)

**ID:** BL-009  
**Category:** BitLocker / Physical Attack Hardening  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1803+, Windows 11  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      └─ Disable new DMA devices when this computer is locked: Enabled

Computer Configuration
  └─ Administrative Templates
       └─ System
            └─ Kernel DMA Protection
                 └─ Enumeration policy for external devices incompatible with
                      Kernel DMA Protection: Block All
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\FVE
  DisableExternalDMAUnderLock = 1   ; Block DMA devices when locked

HKLM\SOFTWARE\Policies\Microsoft\Windows\Kernel DMA Protection
  DeviceEnumerationPolicy = 0   ; 0=block all, 2=allow all (default)
```

## DMA Attack Chain & Mitigations

```
DMA (Direct Memory Access) attacks:
  Tools: PCILeech, inception, MemScout
  Vector: Thunderbolt 3/4, PCIe, FireWire

Attack flow:
  1. Machine is running/sleeping (S3)
  2. Attacker connects PCILeech device via Thunderbolt
  3. PCILeech requests DMA access (PCI bus)
  4. Direct read of physical RAM
  5. Searches for BitLocker FVEK signature in memory
  6. Extracts FVEK → decrypts drive without PIN

Mitigations (layered):
  BL-006: Disable S3 sleep → FVEK not in RAM during lock
  BL-009: DisableExternalDMAUnderLock → no new DMA when locked
  Kernel DMA Protection: IOMMU-based DMA isolation (hardware)
  VT-d / AMD-Vi: Required at hardware/BIOS level

Check if Kernel DMA Protection is active:
  msinfo32.exe → Kernel DMA Protection: On
  Get-SystemInformation (MsSecConfig)
```

## PowerShell — Check DMA Protection Status

```powershell
# Check Kernel DMA Protection status
$sysinfo = Get-ComputerInfo
$sysinfo.DeviceGuardSmartStatus

# Alternative: check via WMI
(Get-WmiObject -Namespace root\Microsoft\Windows\DeviceGuard `
    -Class Win32_DeviceGuard).VirtualizationBasedSecurityStatus

# Check if Thunderbolt DMA blocked
Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Kernel DMA Protection" `
    -Name DeviceEnumerationPolicy -ErrorAction SilentlyContinue
```
