# BL-009 — Disable BitLocker on Standard User Request (Require Admin)

**ID:** BL-009  
**Category:** BitLocker / Administrative Control  
**Risk Level:** 🟠 Medium  
**OS:** Windows 10+, Windows 11  
**Source:** Microsoft BitLocker ADMX

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ [All drive types]
                      └─ Configure use of hardware-based encryption → Enabled

; Prevent users from suspending or turning off BitLocker:
User Configuration (or Computer)
  └─ Restrict access to BitLocker control panel
```

## Registry

```
Key:   HKLM\SOFTWARE\Policies\Microsoft\FVE
Value: DisableExternalDMAUnderLock = 1 — Block DMA when locked
Value: FDVAllowUserCert            = 0 — Prevent user cert for data drives
```

## PowerShell

```powershell
# Check if standard user can manage BitLocker
# (By default, standard users CAN suspend BitLocker — this is a risk)

# Audit BitLocker suspension events:
Get-WinEvent -LogName "Microsoft-Windows-BitLocker/BitLocker Management" | 
    Where-Object { $_.Id -eq 775 } |  # Event 775 = BitLocker suspended
    Select TimeCreated, Message

# Prevent suspension via policy — enforce via SCCM/Intune compliance baseline
# Check protection status:
Get-BitLockerVolume -MountPoint "C:" | Select ProtectionStatus
# ProtectionStatus = 0 (Suspended) or 1 (On)
```

## Description

Prevents standard users from suspending or disabling BitLocker protection. By default, Windows allows standard users to suspend BitLocker temporarily — this is exploited by malware and rogue users to create a window where the drive is unprotected. Restricting BitLocker management to administrators ensures continuous protection.

## Impact

- ✅ Prevents malware from suspending BitLocker protection
- ✅ Prevents users from creating unprotected boot windows
- ⚠️ Users cannot suspend BitLocker for firmware updates (IT must do it)
- ℹ️ Monitor Event ID 775 (BitLocker Suspended) in your SIEM
