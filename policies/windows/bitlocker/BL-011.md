# BL-011 — Enable Secure Boot + TPM Platform Validation for BitLocker

**ID:** BL-011  
**Category:** BitLocker / Platform Integrity / Secure Boot  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1607+, Windows 11 (UEFI required)  
**Source:** Microsoft BitLocker ADMX · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      └─ Allow Secure Boot for integrity validation
                           └─ Enabled
```

## Registry

```
Key:   HKLM\SOFTWARE\Policies\Microsoft\FVE
Value: OSAllowSecureBootForIntegrity
Type:  REG_DWORD
Data:  1
```

## PowerShell

```powershell
# Check Secure Boot status
Confirm-SecureBootUEFI

# Check TPM status
Get-Tpm | Select TpmPresent, TpmReady, TpmEnabled, TpmActivated, ManufacturerId

# Check PCR validation profile (platform validation)
manage-bde -protectors -get C: | findstr "PCR"

# Check BitLocker using Secure Boot
Get-BitLockerVolume C: | Select *
```

## Description

Configures BitLocker to use Secure Boot for OS integrity validation instead of the legacy TPM PCR (Platform Configuration Register) validation profile. Secure Boot-based validation is more reliable across hardware changes and Windows updates — the old PCR-based approach would lock users out after BIOS updates or hardware changes, requiring recovery key entry. This policy provides strong boot integrity validation with lower operational friction.

## Impact

- ✅ Validates boot chain integrity (UEFI → bootloader → OS)
- ✅ Detects bootkit/rootkit modifications before Windows loads
- ✅ More resilient to false lockouts from BIOS updates vs legacy PCR profile
- ⚠️ Requires UEFI firmware (not compatible with Legacy BIOS)
- ⚠️ Secure Boot must be enabled in UEFI settings
