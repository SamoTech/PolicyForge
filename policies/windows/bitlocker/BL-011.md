## PCR (Platform Configuration Register) Explained

```
TPM PCRs store SHA-1/SHA-256 hashes of boot components:

  PCR 0: BIOS/UEFI firmware code
  PCR 1: BIOS/UEFI configuration data
  PCR 2: Option ROM code (expansion cards)
  PCR 3: Option ROM configuration
  PCR 4: Master Boot Record + boot manager code
  PCR 5: MBR partition table
  PCR 6: State transition events
  PCR 7: Secure Boot state (UEFI only)
  PCR 8-9: NTLDR, bootmgr
  PCR 11: BitLocker access control

BitLocker seals FVEK against PCR values at provisioning time.
If ANY selected PCR changes at boot → TPM refuses to unseal FVEK
→ BitLocker enters recovery mode
```

## Recommended PCR Configuration

| PCR Set | Validates | Use Case |
|---|---|---|
| PCR 0, 2, 4, 11 | BIOS, option ROMs, MBR, BL access | Standard laptops |
| PCR 7, 11 | Secure Boot + BL access only | Flexible UEFI systems |
| PCR 0, 2, 4, 5, 11 | + Partition table | High-security desktops |

## Evil Maid Attack Context

```
Evil Maid attack (targeted physical attack):
  1. Attacker gains brief physical access to locked laptop
  2. Boots from USB → modifies bootloader/MBR
  3. Modified bootloader captures BitLocker PIN
  4. Attacker retrieves device later, enters captured PIN
  5. Full disk access

With PCR 4 measurement (MBR/bootloader):
  Step 2 → Modified bootloader changes PCR 4 value
  Step 4 → TPM detects PCR mismatch → REFUSES to release FVEK
  → BitLocker enters recovery mode
  → Attack fails (cannot enter PIN to unlock)
  → PIN capture attempt is detected
```
