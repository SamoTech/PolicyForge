# BL-010 — Disable DMA Access When Screen is Locked

**ID:** BL-010  
**Category:** BitLocker / DMA Attack Prevention  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10 1803+, Windows 11  
**Source:** Microsoft Security Baseline · CIS Benchmark

---

```
Computer Configuration
```

## Description

Blocks new DMA (Direct Memory Access) devices from connecting when the screen is locked. Thunderbolt, PCIe, and FireWire devices with DMA capability can directly read system memory — bypassing the OS and extracting BitLocker encryption keys, session tokens, or credentials from RAM. This attack (PCILeech, Inception) works even on BitLocker-protected systems if the attacker has physical access to a DMA port on a running locked system.

## Impact

- ✅ Blocks DMA-based memory extraction attacks (PCILeech, Inception)
- ✅ Prevents Thunderbolt hot-plug exploitation when locked
- ⚠️ Hot-plugged Thunderbolt docks/eGPUs won't work when screen is locked
- ℹ️ Kernel DMA protection (IOMMU/VT-d) should also be enabled for deeper protection
