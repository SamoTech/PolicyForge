# BL-006 — Disable Hibernate & Sleep for BitLocker Protection

**ID:** BL-006  
**Category:** BitLocker / Sleep State Security  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11  
**Source:** CIS Benchmark Windows 11 v3.0 · DISA STIG · Microsoft Security Baseline

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ System
            └─ Power Management
                 └─ Sleep Settings
                      ├─ Allow standby states (S1-S3) when sleeping (on battery): Disabled
                      ├─ Allow standby states (S1-S3) when sleeping (plugged in): Disabled
                      └─ Require a password when a computer wakes (plugged in): Enabled
                           Require a password when a computer wakes (on battery): Enabled
```

## The Sleep State Attack (DMA + Cold Boot)

```
Sleep/Standby (S3) vulnerability:
  When Windows sleeps (S3), DRAM contents remain powered
  BitLocker FVEK remains in RAM
  → DMA attack via Thunderbolt/PCIe:
     Tool: PCILeech, MemScout, inception
     → Direct memory read via DMA
     → Extract BitLocker FVEK from RAM
     → Decrypt drive without PIN
  → Cold boot attack:
     → Freeze RAM with liquid nitrogen/compressed air
     → Transfer frozen RAM modules to attacker system
     → Read FVEK before bits decay (~seconds to minutes)

Hibernate (S4) vulnerability:
  Hibernation writes full RAM contents to hiberfil.sys
  → Hiberfil.sys is on the C: drive
  → If BitLocker not yet unlocked at boot: C: is encrypted
  ✅ Hibernate is SAFE when combined with TPM+PIN
  ✔ Windows resumes from hiberfil.sys → requires PIN

S3 sleep is UNSAFE regardless of PIN (FVEK in RAM)
Best practice: Disable S3, use S4 (hibernate) or shutdown
```

## PowerShell

```powershell
# Disable all sleep states (use hibernate or shutdown only)
powercfg /hibernate on
powercfg /change standby-timeout-ac 0
powercfg /change standby-timeout-dc 0

# Require password on wake
powercfg /SETACVALUEINDEX SCHEME_CURRENT SUB_NONE CONSOLELOCK 1

# Check current sleep states
powercfg /availablesleepstates
```
