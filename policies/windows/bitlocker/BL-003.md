# BL-003 — BitLocker Recovery Key Management

**ID:** BL-003  
**Category:** BitLocker / Recovery Key Management  
**Risk Level:** 🔴 Critical  
**OS:** Windows 10+, Windows 11, Server 2016+  
**Source:** CIS Benchmark Windows 11 v3.0 · Microsoft Security Baseline · DISA STIG

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Windows Components
            └─ BitLocker Drive Encryption
                 └─ Operating System Drives
                      ├─ Choose how BitLocker-protected operating system drives
                      │    can be recovered: Enabled
                      │    ├─ Allow data recovery agent: Enabled
                      │    ├─ Configure user storage of BitLocker recovery information:
                      │    │    Do not allow 48-digit recovery password
                      │    ├─ Omit recovery options from the BitLocker setup wizard: Enabled
                      │    ├─ Save BitLocker recovery information to AD DS: Enabled
                      │    ├─ Configure storage of BitLocker recovery information to AD DS:
                      │    │    Store recovery passwords and key packages
                      │    └─ Do not enable BitLocker until recovery information
                      │         is stored in AD DS: Enabled
                      └─ (Entra ID / Intune): Backup recovery key to Azure AD: Enabled
```

## Registry

```
HKLM\SOFTWARE\Policies\Microsoft\FVE
  OSRecovery                    = 1   ; Enable recovery options
  OSManageDRA                   = 1   ; Allow data recovery agent
  OSRecoveryPassword            = 0   ; Do not allow user-generated recovery password
  OSRecoveryKey                 = 0   ; Do not allow user-generated recovery key
  OSRequireActiveDirectoryBackup = 1  ; Require AD backup before enabling
  OSActiveDirectoryBackup       = 1   ; Save to AD DS
  OSActiveDirectoryInfoToStore  = 2   ; Store passwords + key packages
  OSHideRecoveryPage            = 1   ; Hide recovery options from setup wizard
```

## PowerShell — Recovery Key Management

```powershell
# Backup existing BitLocker recovery key to AD
$vol = Get-BitLockerVolume -MountPoint C:
$recoveryKeyId = ($vol.KeyProtector | Where-Object {$_.KeyProtectorType -eq 'RecoveryPassword'}).KeyProtectorId
Backup-BitLockerKeyProtector -MountPoint C: -KeyProtectorId $recoveryKeyId

# Backup to Azure AD (Entra ID) via Intune
BackupToAAD-BitLockerKeyProtector -MountPoint C: -KeyProtectorId $recoveryKeyId

# View recovery key (local - admin only)
(Get-BitLockerVolume C:).KeyProtector | Where-Object {$_.KeyProtectorType -eq 'RecoveryPassword'} |
    Select RecoveryPassword

# Retrieve from AD (helpdesk)
$computer = Get-ADComputer -Identity "WORKSTATION01" -Properties *
Get-ADObject -Filter {objectclass -eq 'msFVE-RecoveryInformation'} `
    -SearchBase $computer.DistinguishedName -Properties msFVE-RecoveryPassword
```

## Recovery Key Security Rules

```
✅ Always: Store recovery key in AD DS or Entra ID BEFORE enabling BitLocker
✅ Always: Require AD backup before BitLocker can be enabled (GPO enforced)
✅ Always: Store key packages (not just passwords) for forensic recovery
✅ Always: Rotate recovery keys after use (key may be compromised)
❌ Never: Store recovery key on the encrypted drive itself
❌ Never: Store recovery key in a network share on the same machine
❌ Never: Print recovery key and keep it with the device
❌ Never: Allow users to manage their own recovery keys
```
