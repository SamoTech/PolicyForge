# NET-002 — Disable NetBIOS over TCP/IP

**ID:** NET-002
**Category:** Network Hardening
**Risk Level:** 🔴 Critical
**OS:** Windows 7+, Windows 11, Windows Server 2008+
**Source:** Windows 11 Security Baseline Hardening Guide 2026 · [osmachine.com](https://osmachine.com/public/index.php/blog/windows-11-security-baseline-hardening-guide)

## Registry

```
Key:   HKLM\SYSTEM\CurrentControlSet\Services\NetBT\Parameters\Interfaces\Tcpip_{GUID}
Value: NetbiosOptions
Type:  DWORD
Data:  2  (0=default/DHCP, 1=enabled, 2=disabled)
```

## PowerShell

```powershell
# Disable NetBIOS on ALL network adapters
$adapters = Get-WmiObject Win32_NetworkAdapterConfiguration | Where-Object { $_.IPEnabled }
foreach ($adapter in $adapters) {
    $adapter.SetTcpipNetbios(2) | Out-Null
    Write-Host "Disabled NetBIOS on: $($adapter.Description)"
}

# Verify
Get-WmiObject Win32_NetworkAdapterConfiguration | Where-Object { $_.IPEnabled } |
    Select Description, TcpipNetbiosOptions
```

## Description

NetBIOS over TCP/IP (NBT) is a legacy name resolution protocol from the 1980s that is actively exploited by tools like Responder to poison name resolution and capture Net-NTLMv2 hashes. Disabling it removes the NBT broadcast attack surface entirely. Azure AD / Entra ID joined devices should always have this disabled.

## Impact

- ✅ Eliminates NBT-NS poisoning attack vector (Responder, Inveigh)
- ✅ Reduces broadcast traffic on the network
- ⚠️ Breaks legacy applications relying on NetBIOS name resolution
- ⚠️ May break printing to very old printers using NetBIOS discovery
- ℹ️ Modern environments using DNS are unaffected

## Use Cases

- All modern domain-joined environments using DNS
- Azure AD / Entra ID joined devices
- Environments running red team exercises (removes easy hash capture)
- Workstations on untrusted network segments
