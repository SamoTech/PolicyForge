# NET-003 — Disable LLMNR (Link-Local Multicast Name Resolution)

**ID:** NET-003
**Category:** Network Hardening
**Risk Level:** 🔴 Critical
**OS:** Windows Vista+, Windows 11, Windows Server 2008+
**Source:** Windows 11 Security Baseline Hardening Guide 2026 · [osmachine.com](https://osmachine.com/public/index.php/blog/windows-11-security-baseline-hardening-guide)

```
Key:   HKLM\Software\Policies\Microsoft\Windows NT\DNSClient
Value: EnableMulticast

Data:  0
```

## PowerShell

```powershell
# Disable LLMNR via registry
$path = "HKLM:\Software\Policies\Microsoft\Windows NT\DNSClient"
If (-not (Test-Path $path)) { New-Item -Path $path -Force }
Set-ItemProperty -Path $path -Name "EnableMulticast" -Value 0 -Type DWord

# Verify
Get-ItemProperty -Path $path -Name EnableMulticast
```

## Description

LLMNR is a protocol that resolves single-label hostnames on the local link when DNS fails. Attackers use tools like **Responder** to answer LLMNR queries with a rogue IP, capturing NTLMv2 hashes for offline cracking or relay attacks. Disabling LLMNR is one of the highest-impact, lowest-effort hardening steps for any Windows environment.

## Impact

- ✅ Eliminates the #1 most commonly abused name poisoning vector
- ✅ Breaks Responder, Inveigh, and Metasploit auxiliary/spoof/llmnr/nbns_response
- ⚠️ Machines must rely on DNS for all name resolution — mistyped hostnames will fail
- ℹ️ Zero impact in properly configured DNS environments
