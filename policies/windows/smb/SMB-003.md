# SMB-003 — Enable SMB Signing (Client)

**ID:** SMB-003
**Category:** Network / SMB Hardening
**Risk Level:** 🔴 Critical

## PowerShell

```powershell
# Require SMB signing on the client
Set-SmbClientConfiguration -RequireSecuritySignature $true -Force

# Enable (but not require) as a softer option
Set-SmbClientConfiguration -EnableSecuritySignature $true -Force

# Verify current state
Get-SmbClientConfiguration | Select EnableSecuritySignature, RequireSecuritySignature
```

## Description

Requires all SMB client communications to be digitally signed. SMB signing authenticates each packet, preventing tampering and relay attacks. This is a foundational control against NTLM relay and Pass-the-Hash attacks. Required by the Microsoft Security Baseline since Windows 11 23H2.

## Impact

- ✅ Prevents NTLM relay attacks using SMB
- ✅ Authenticates every SMB packet against tampering
- ⚠️ Breaks connections to legacy SMB servers with signing disabled
- ⚠️ ~3-5% CPU overhead on file-intensive workloads (negligible with modern CPUs)

## Use Cases

- All domain-joined environments (baseline requirement)
- Environments running Responder/NTLM relay mitigation
- Zero-trust internal network segments
- Post-Active Directory compromise hardening
