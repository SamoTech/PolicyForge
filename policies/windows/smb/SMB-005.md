# SMB-005 — Set Maximum SMB Dialect to SMB 3.1.1

**ID:** SMB-005
**Category:** Network / SMB Hardening
**Risk Level:** 🟠 Medium
**OS:** Windows 11 24H2+, Windows Server 2025
**Source:** Windows 11 24H2 new ADMX SMB settings · [4sysops.com](https://4sysops.com/archives/windows-11-24h2-group-policy-81-new-settings-for-smb-updates-printing-defender-and-more/)

---

## Policy Path

```
Computer Configuration
  └─ Administrative Templates
       └─ Network
            └─ Lanman Workstation
                 └─ Minimum SMB 2 Dialect (set maximum via PowerShell)
```

## Registry

```
Key:   HKLM\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters
Value: MaxSmb2Dialect
Type:  DWORD
Data:  0x0311  (785 decimal = SMB 3.1.1)
```

## PowerShell

```powershell
# Set minimum dialect to SMB 3.0 (disallow SMB 1 and 2)
Set-SmbClientConfiguration -MinimumDialect SMB30 -Force

# Block SMB 1 entirely (separate control)
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
Set-SmbClientConfiguration -EnableBandwidthThrottling $false

# Check current dialect negotiation
Get-SmbConnection | Select ServerName, Dialect
```

```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters]
"MinSmb2Dialect"=dword:00000300
```

## Description

New in Windows 11 24H2 ADMX templates — allows administrators to enforce a minimum SMB dialect, effectively blocking connections that downgrade to SMB 2.0 or SMB 2.1. Combined with SMB-001 (encryption) and SMB-003 (signing), this closes dialect downgrade attack vectors.

## Impact

- ✅ Prevents dialect downgrade attacks
- ✅ Enforces modern SMB 3.x feature set (encryption, signing, pre-auth integrity)
- ⚠️ Breaks connections to Windows 7 / Server 2008 R2 (SMB 2.1 max)
- ⚠️ Breaks connections to old NAS devices with SMB 2.x only firmware

## Use Cases

- Modern enterprise environments (Windows 10 1703+ clients only)
- Environments decommissioning legacy file servers
- Post-migration hardening after SMB 1 removal
