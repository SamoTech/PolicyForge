## Description

Requires all inbound SMB connections to the Windows file server to be digitally signed. This is the server-side counterpart to SMB-003. Both client and server signing must be enabled for end-to-end signing. Default on Domain Controllers since Windows Server 2025; should be explicitly enforced on all member servers.

## Impact

- ✅ Prevents rogue clients from connecting without authentication
- ✅ Blocks NTLM relay attacks targeting the server
- ⚠️ Rejects connections from clients with signing disabled
- ⚠️ Required on DCs by default — may already be set

## Use Cases

- File servers in AD environments
- Print servers accepting SMB connections
- Any server sharing files via SMB protocol
